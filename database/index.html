Databases & SQL — The Full Course

Fifty-five self-contained, interactive lessons on relational databases — taught through PostgreSQL, the default choice for new systems. The relational model and schema design first, then SQL from joins to window functions, then the parts that decide whether a system survives contact with real load: indexing, the query planner, transactions and MVCC, single-node operations — and a full phase on running Postgres as a distributed system: replication, consistency under lag and failure, and keeping caches and search in step with the database of record. Every lesson runs the same way — a clear mental model, schematic play-through diagrams, a worked SQL problem, and a self-test.

Original study material. Built from scratch for engineers who want database depth that pays off in both interviews and production. Examples target a recent PostgreSQL (17/18); the core ideas — the relational model, ACID, indexing, MVCC, replication — are general, and where another database differs it is called out. Every explanation, diagram, query, and line of code is original.
How to use it

Do L00–L03 first — what a database guarantees and how Postgres is built underpin everything else. Then walk the phases in order; indexing (P4) and transactions & MVCC (P5) are the heart of the fundamentals, and Distributed PostgreSQL (P8) is where they pay off at scale — replication, read-after-write, failover, the dual-write problem, and sharding. The guiding rule throughout: make it correct first — with the right schema and constraints — then make it fast, and always read the query plan before you guess.
