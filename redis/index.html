Redis — The Full Course

Forty-five self-contained, interactive lessons on Redis — from the in-memory, single-threaded core and the handful of data structures that define it, through caching strategy, persistence, replication and Cluster, to the modern platform layer where one server also does search, vector similarity, and LLM memory. Organised bottom-up and dependency-aware: the data-structure server first, then the patterns built on top of it, how it scales and stays alive, the Redis 8 platform, and the operational and interview concerns. Every lesson runs the same way — a clear mental model, schematic play-through diagrams, worked commands, and a self-test.

Original study material. Built from scratch for engineers preparing for backend, infrastructure, and platform interviews — and for the real work behind them. The framing draws on Redis's official documentation and command reference for accuracy, but every explanation, diagram, and line of code is original. Version and feature details track the Redis 8.x line and are current as of mid-2026 — including Redis 8 folding the former Stack modules (Query Engine, JSON, Time Series, probabilistic, vector) into core, the newer vector sets type, and the AGPLv3 relicensing — and are treated as illustrative throughout, because the project moves fast.
How to use it

Do L00–L08 first — the in-memory, single-threaded model and the five core data types (strings, hashes, lists, sets, sorted sets) underpin everything else. Then walk the phases in order. The guiding rule throughout: reach for a richer structure or a bigger topology only when you can name why the simpler one fell short — a sorted set before a hand-rolled index, a Lua script before a client-side lock, Cluster before you've proven one node won't do. Short on time? The ★ Core 15 are the highest-yield path.
