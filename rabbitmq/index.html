RMQThe Message Broker · Interactive Course

RabbitMQ — The Full Course

Forty-five self-contained, interactive lessons on RabbitMQ and message-driven architecture — from the producer / exchange / queue model and the routing types to delivery guarantees, quorum-queue high availability, log-based streams, the multi-protocol broker, and running a cluster in production. Organised bottom-up and dependency-aware: the AMQP model first, then routing, queues, and the reliability machinery, then scale, streams, and the operational concerns. Every lesson runs the same way — a clear mental model, schematic play-through diagrams, worked examples, and a self-test.

Original study material. Built from scratch for engineers preparing for backend, platform, and distributed-systems interviews — and for the real work behind them. The framing draws on RabbitMQ's official documentation for accuracy, but every explanation, diagram, and line of code is original. Version and feature details track the RabbitMQ 4.x line — the Khepri metadata store, native AMQP 1.0, and quorum-queue HA — and are current as of mid-2026, treated as illustrative throughout, because the broker moves fast.
How to use it

Do L00–L07 first — the broker model and the AMQP entities (connection, channel, exchange, binding, queue, consumer) underpin every later phase. Then walk the phases in order. The guiding rule throughout: reach for the broker only when a direct synchronous call genuinely fails you — and inside it, reach for the heavier primitive only when you can name why the simpler one breaks: a quorum queue over classic only for data you can't lose, a stream over a queue only when you need replay. Short on time? The ★ Core 15 are the highest-yield path.
