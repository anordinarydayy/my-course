Subsequence DP

A subsequence keeps the original order but is allowed to skip elements. That freedom makes the search space exponential — yet a well-chosen DP table collapses it to polynomial time. This lesson covers the two shapes every interviewer reaches for: single-sequence tables (longest increasing run) and two-sequence grids (common characters, edit operations).

00 The frame

What "subsequence" means and why DP fits

A subsequence of a sequence is any selection of elements taken in the same relative order, with arbitrary gaps allowed. The subsequences of [A, B, C] include [A, C], [B], and [A, B, C] itself, but not [C, A] — that reverses the order. For a sequence of length n, there are 2n possible subsequences: trying them all is hopeless at n = 40.

Dynamic programming fits because the problem has overlapping subproblems and optimal substructure. Whether the last element of the array is part of the best answer depends only on what the best answer looks like for the elements before it — a smaller version of the same question. Two shapes dominate interview problems:

  • One-sequence tables — dp[i] holds the answer for the prefix ending at index i. The final answer is a scan over all entries (not necessarily dp[n-1]). Classic: Longest Increasing Subsequence.
  • Two-sequence grids — dp[i][j] holds the answer for the prefix of length i from one sequence and length j from another. Alignment at the corner element drives the recurrence. Classics: Longest Common Subsequence, Edit Distance.
The reusable shape

For both shapes the pattern is identical: define what dp[i] (or dp[i][j]) means precisely, write the recurrence, identify the base cases, and read off the answer. The interview question is usually whether you can derive the recurrence from first principles — so always state it out loud before touching the keyboard.

01 One-sequence

Longest Increasing Subsequence — O(n²) DP

The brute force generates every 2n subsequence and filters for increasing ones — workable at n = 20, catastrophic at n = 2500. The waste is clear: the same prefix is re-examined exponentially many times. DP fixes this by asking a smaller question: what is the longest increasing subsequence that ends specifically at index i?

Define dp[i] as the length of the LIS whose last element is a[i]. For every j < i, if a[j] < a[i] then we can extend that subsequence: dp[i] = max(dp[i], dp[j] + 1). The base case is dp[i] = 1 for all i (every single element is a length-1 subsequence). The final answer is max(dp) — not dp[n-1], because the LIS might end anywhere.

This is O(n²) time, O(n) space. The quadratic scan of all prior indices is the next bottleneck — section 02 removes it with a patience-sort trick.

Worked problemMediumLeetCode 300
Longest Increasing Subsequence

Given an integer array, find the length of its longest strictly increasing subsequence — elements can be non-contiguous, but their relative order must be preserved.

Think it through

Restate

Pick the most elements from the array such that each picked element is strictly larger than the one before it (in original order). Return the count.

Brute force

Try every subset: 2n subsets, each checked in O(n). That is O(n · 2n) — impossible for n = 2500.

The waste

The same prefix is checked again and again across different subsets. Memoize it: dp[i] = length of LIS ending at index i, computed once.

The move

Fill left to right. For each i, scan every j < i: if a[j] < a[i], we can extend that chain, so dp[i] = max(dp[i], dp[j] + 1). Start every dp[i] = 1. Answer is max(dp).

Invariant & cost

When computing dp[i], every dp[j] for j < i is already final. O(n²) time, O(n) space. The binary-search variant (section 02) brings this to O(n log n).

The code

python

def lengthOfLIS(nums):
    n = len(nums)
    if n == 0:
        return 0
    dp = [1] * n          # every element alone is length 1
    for i in range(1, n):
        for j in range(i):
            if nums[j] < nums[i]:    # can extend the chain at j
                dp[i] = max(dp[i], dp[j] + 1)
    return max(dp)
Narrate it to the interviewer

"I need the longest strictly increasing subsequence. Brute force is exponential — 2-to-the-n subsets. The fix is DP: I define dp[i] as the LIS length whose last element is a[i]. I fill left to right; for each i I look back at every j less than i, and if a[j] is smaller I can extend that chain, so dp[i] becomes the max of itself and dp[j] plus 1. Every dp[i] starts at 1. The invariant is that when I compute dp[i], everything to the left is already solved. The answer is the max of the whole dp array — not dp[n-1], because the LIS can end anywhere. Time O(n²), space O(n)."

Dry-run aloud: empty array (return 0), single element (dp = [1], return 1), strictly decreasing array (every dp[i] stays 1, answer = 1), all equal (strictly less fails every check, answer = 1).

02 One-sequence, faster

LIS in O(n log n) — patience sorting with a tails array

The quadratic scan in section 01 asks: "which previously computed dp values can I extend?" There is a smarter question: what is the smallest possible tail element of any increasing subsequence of each length? Tracking that turns each step into a binary search.

Maintain a tails array T where T[k] is the smallest tail value among all increasing subsequences of length k + 1 seen so far. For each new element x from the input:

  • Binary-search T for the leftmost position p where T[p] ≥ x.
  • If no such position exists (x is larger than everything), append x — the LIS just grew by one.
  • Otherwise, replace T[p] with x — a subsequence of the same length now has a smaller tail, making future extensions easier.

The length of T at the end equals the LIS length. Crucially, T is always sorted (invariant), so binary search is valid. Note that T itself is not the actual LIS — it is a compressed ledger of achievable tail values. Time: O(n log n); space: O(n).

Why replacing does not corrupt the answer

Replacing T[p] with a smaller value x means: there now exists an increasing subsequence of length p+1 whose last element is x. That is strictly better than the old tail — it leaves more room for future elements to extend the chain. The length of T (not its contents) is what gives the LIS length, and that length only ever grows by 1 at a time (on an append), which is correct.

O(n²) DP — dp[i]

  • Simple to derive and explain.
  • Can reconstruct the actual subsequence by tracing back a parent array.
  • Quadratic — too slow for n > ~5000.

O(n log n) — tails + bisect

  • Works for n up to 105 or more.
  • Reconstruction possible but needs extra bookkeeping.
  • Requires understanding why T stays sorted and why replace is safe.
What the interviewer asks next

"Can you reconstruct the actual subsequence, not just its length?" Yes — add a parent[] array alongside dp (for the O(n²) version) or record which index was replaced at each step (for tails). Walk back from the last element of the LIS. "What if the condition is non-decreasing instead of strictly increasing?" Use bisect_right instead of bisect_left in the tails approach — duplicates are then allowed to extend a chain.

03 Two-sequence

Longest Common Subsequence — filling the 2-D grid

Now bring in a second sequence. The LCS of two strings is the longest sequence of characters that appears (in order, with gaps allowed) in both. For "ABCDE" and "ACE" the LCS is "ACE" of length 3. The brute force checks every subsequence of the shorter string against the longer one — exponential in the shorter length.

DP instead asks: what is the LCS of the prefix of length i of the first string and the prefix of length j of the second? Call that dp[i][j]. The recurrence has two cases:

  • If s1[i-1] == s2[j-1] (last characters match): dp[i][j] = dp[i-1][j-1] + 1. The match extends the LCS of the two shorter prefixes — move diagonally.
  • Otherwise: dp[i][j] = max(dp[i-1][j], dp[i][j-1]). The best LCS comes from either skipping the current character of s1 (look up) or skipping the current character of s2 (look left).

Base cases: dp[0][j] = 0 and dp[i][0] = 0 — the LCS with an empty string is always zero. Fill row by row. The answer is dp[m][n]. Time: O(m·n); space: O(m·n) (reducible to O(min(m,n)) with rolling rows).

Worked problemMediumLeetCode 1143
Longest Common Subsequence

Given two strings, return the length of their longest common subsequence — the longest sequence of characters, in order, that appears (with gaps allowed) in both strings. Both strings can be up to 1000 characters long.

Think it through

Restate

Find the longest sequence of characters that is a subsequence of both inputs simultaneously. Return its length.

Brute force

Enumerate all subsequences of the shorter string, check each against the other. If the shorter string has length m, that's O(2m · n) — exponential.

The waste

Subproblems repeat: "what is the LCS of s1[0..i] and s2[0..j]?" is computed many times during recursion. Memoize it into a 2-D grid.

The move

Build a (m+1)×(n+1) grid, row 0 and column 0 all zeros. For each cell (i, j): if s1[i-1] == s2[j-1], take the diagonal plus one; otherwise take the max of the cell above and the cell to the left. Return dp[m][n].

Invariant & cost

dp[i][j] holds the LCS of the first i characters of s1 and first j of s2. When we read dp[i-1][j-1], dp[i-1][j], dp[i][j-1] they are all already final (filled in earlier rows/columns). O(m·n) time, O(m·n) space — reducible to O(min(m,n)) by keeping only two rows.

The code

python

def longestCommonSubsequence(s1, s2):
    m, n = len(s1), len(s2)
    # (m+1) x (n+1) grid; row 0 and col 0 are implicitly 0
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s1[i - 1] == s2[j - 1]:   # characters match → diagonal + 1
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:                          # skip one char from either string
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])
    return dp[m][n]
Narrate it to the interviewer

"The brute force is exponential — too many subsequences. I'll use a 2-D DP grid where dp[i][j] is the LCS of the first i characters of s1 and first j of s2. Base case: the first row and column are all zero — the LCS with an empty string is zero. For each cell, if the current characters match I take the diagonal plus one; otherwise I take the max of the cell above or to the left, representing skipping a character from one string or the other. The final answer is in the bottom-right corner. Time O(m·n), space O(m·n). If asked to save space, I can keep only the previous row, cutting space to O(n)."

Dry-run aloud: one or both strings empty (return 0), identical strings (LCS = length of string, every diagonal fires), completely disjoint characters (every cell filled via max of up/left, result is 0).

04 Two-sequence, operations

Edit Distance — insert, delete, replace

LCS asks how much two strings share. Edit Distance asks how much they differ — formally, the minimum number of single-character operations (insert, delete, replace) needed to transform one string into another.

Define dp[i][j] as the edit distance between the first i characters of s1 and the first j characters of s2. The recurrence:

  • If s1[i-1] == s2[j-1]: no operation needed at this position — dp[i][j] = dp[i-1][j-1].
  • Otherwise: take the minimum of three neighbors plus one operation — dp[i][j] = 1 + min(dp[i-1][j-1], dp[i-1][j], dp[i][j-1]), representing replace, delete from s1, and insert into s1 respectively.

Base cases: dp[i][0] = i (delete all i characters) and dp[0][j] = j (insert j characters). Fill row by row. Answer: dp[m][n]. Time O(m·n), space O(m·n) (or O(n) with rolling rows).

Worked problemMediumLeetCode 72
Edit Distance

Given two strings, find the minimum number of single-character operations (insert one character, delete one character, or replace one character) needed to turn the first string into the second.

Think it through

Restate

The edit distance (Levenshtein distance) between two strings. Every operation costs 1. Find the minimum total cost to transform s1 into s2.

Brute force

Recursively try all three operations at each mismatch. The recursion tree has branching factor 3 at each step — O(3m+n) without memoization.

The waste

Subproblems repeat exponentially: "what is the edit distance of s1[i:] and s2[j:]?" is recomputed at every branch. A 2-D DP table computes each subproblem once.

The move

Fill a (m+1)×(n+1) table. Base cases: dp[i][0] = i, dp[0][j] = j. For inner cells: if characters match, copy the diagonal; if not, take 1 + min(diagonal, above, left) — representing replace, delete, and insert respectively.

Invariant & cost

dp[i][j] is the minimum operations to convert s1[:i] to s2[:j]. The three neighbors it reads (dp[i-1][j-1], dp[i-1][j], dp[i][j-1]) are all finalized before (i,j) is computed. O(m·n) time, O(m·n) space; rolling rows reduce space to O(n).

The code

python

def minDistance(s1, s2):
    m, n = len(s1), len(s2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(m + 1): dp[i][0] = i  # delete all of s1
    for j in range(n + 1): dp[0][j] = j  # insert all of s2
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s1[i - 1] == s2[j - 1]:         # match: no operation needed
                dp[i][j] = dp[i - 1][j - 1]
            else:                                # replace / delete / insert
                dp[i][j] = 1 + min(
                    dp[i - 1][j - 1],  # replace
                    dp[i - 1][j],      # delete from s1
                    dp[i][j - 1]       # insert into s1
                )
    return dp[m][n]
Narrate it to the interviewer

"I need the minimum edits to transform s1 into s2 using insert, delete, or replace. Brute-force recursion is exponential. I'll use a 2-D DP table where dp[i][j] is the minimum edits to convert the first i characters of s1 into the first j of s2. Base cases: dp[i][0] = i because I delete everything; dp[0][j] = j because I insert everything. For each inner cell: if the current characters match, I copy the diagonal — no extra cost. If they don't, I take one plus the minimum of the diagonal (replace), the cell above (delete a character from s1), or the cell to the left (insert a character). The answer is in the bottom-right corner. O(m·n) time and space. If memory is tight I only need the previous row."

Dry-run aloud: one empty string (answer equals length of the other — the base cases handle this), identical strings (every character matches, every cell copies the diagonal, answer is 0), completely different strings (all mismatches, fills like a staircase).

05 Teaser

Distinct Subsequences — counting, not optimizing

The two-sequence grid generalizes beyond lengths and costs. Distinct Subsequences (LeetCode 115) asks: in how many distinct ways can the characters of string t appear as a subsequence inside s? This is a counting problem, not an optimization — but the same grid structure applies.

Define dp[i][j] as the number of distinct ways t[0..j-1] appears as a subsequence of s[0..i-1]. The recurrence:

  • Always include the contribution of not using s[i-1]: dp[i][j] += dp[i-1][j] (skip this character of s).
  • If s[i-1] == t[j-1], also include the ways where we match this pair: dp[i][j] += dp[i-1][j-1].

Base cases: dp[i][0] = 1 for all i (empty t is always a subsequence of anything, in exactly one way — take nothing), and dp[0][j] = 0 for j > 0 (can't match a non-empty t from an empty s). Answer: dp[m][n]. Time O(m·n), space O(m·n).

The counting vs optimizing distinction

LCS and Edit Distance maximize or minimize a quantity, so the recurrence takes max or min. Distinct Subsequences counts, so the recurrence adds. Same grid skeleton, different operator. When you see "how many ways" in a two-sequence problem, reach for addition; when you see "longest" or "minimum cost", reach for max/min. The base case logic follows the same logic — get the zero-row and zero-column right, then let the recurrence do the work.

What the interviewer asks next

"Can you reduce the space from O(m·n) to O(n) for any of these?" Yes — for LCS, Edit Distance, and Distinct Subsequences, you only ever read the previous row and the current row, so two rows suffice. For Distinct Subsequences specifically you need to traverse the current row right-to-left to avoid using updated values. "How does Edit Distance relate to LCS?" The edit distance with only insert and delete equals m + n − 2 · LCS(s1, s2). Replace adds a third neighbor to the recurrence.

06 Practice

Practice ladder

Subsequence DP is a pattern where the recurrence is the whole insight — memorizing a template is much less useful than being able to derive the recurrence from scratch. For each problem below, spend 30 seconds defining dp[i] or dp[i][j] precisely before writing a single line of code.

EasyMaximum Subarray1-D dp · Kadane

Find the contiguous subarray within an integer array that has the largest sum. Return that sum.

Idea dp[i] = max subarray sum ending at index i. Either extend the previous subarray or start fresh: dp[i] = max(dp[i-1] + a[i], a[i]). Answer: max(dp). O(n) time, O(1) space with a rolling variable.

EasyIs Subsequencetwo-pointer / 1-D

Decide whether string t appears as a subsequence of string s — characters of t must appear in s in order, but need not be adjacent.

Idea Two-pointer scan: advance a pointer through t whenever its character matches the current character of s. O(n) time, O(1) space. Good warm-up before tackling LCS.

MediumLongest Increasing SubsequenceLIS — O(n²) dp

Given an integer array, return the length of its longest strictly increasing subsequence.

Idea dp[i] = LIS length ending at index i. Scan all j < i where a[j] < a[i]. Answer = max(dp). O(n²) time; section 01 has the full walkthrough.

MediumNumber of Longest Increasing SubsequencesLIS with count

Return the count of distinct longest increasing subsequences in an integer array.

Idea Maintain two arrays: dp[i] (LIS length ending at i) and cnt[i] (number of such LIS). When dp[j]+1 ties dp[i], add cnt[j] to cnt[i]. O(n²) time.

MediumLongest Common Subsequence2-D grid

Given two strings, find the length of their longest common subsequence — characters in order, gaps allowed.

Idea dp[i][j]: match → diag+1, else max(up, left). Answer at dp[m][n]. O(m·n) time. Section 03 has the interactive grid.

MediumLongest Palindromic SubsequenceLCS variant

Find the longest subsequence of a string that is a palindrome. Return its length.

Idea LCS of the string with its reverse. Or direct DP: dp[i][j] = longest palindromic subsequence in s[i..j]; if s[i]==s[j] then dp[i-1][j-1]+2, else max(dp[i+1][j], dp[i][j-1]). O(n²) time and space.

MediumUncrossed LinesLCS in disguise

Draw lines connecting equal numbers in two integer arrays, without crossing. Return the maximum number of connecting lines.

Idea Exactly LCS on two integer arrays — the order-preserving match is the non-crossing constraint. Identical code to LCS with integer comparison.

MediumEdit Distance2-D, insert/delete/replace

Given two words, find the minimum operations (insert, delete, replace a single character) to convert one word into the other.

Idea dp[i][j] = min edits for prefixes of length i and j. Match → copy diagonal; mismatch → 1 + min(diag, up, left). Base: dp[i][0]=i, dp[0][j]=j. Section 04 walks through this fully.

MediumDelete Operation for Two StringsLCS → min deletes

Given two strings, return the minimum number of characters you must delete (from either string) so the two strings become equal.

Idea The characters you keep must be a common subsequence. Answer = m + n − 2·LCS(s1, s2). O(m·n) time.

HardDistinct Subsequences2-D counting

Given strings s and t, count the number of distinct subsequences of s that equal t.

Idea dp[i][j] += dp[i-1][j] always (skip s[i-1]); if s[i-1]==t[j-1] also add dp[i-1][j-1]. Base: dp[i][0]=1. Answer: dp[m][n]. Section 05 has the explanation.

HardRussian Doll EnvelopesLIS in 2D

You have envelopes with (width, height). One can be nested inside another only if both dimensions are strictly smaller. Find the maximum chain length.

Idea Sort by width ascending, then by height descending (so equal-width envelopes can't form a chain). Run LIS on heights. O(n log n) using the tails array from section 02.

HardShortest Common SupersequenceLCS → build

Find the shortest string that has both given strings as subsequences. Return its length (or reconstruct the string itself).

Idea Length = m + n − LCS(s1, s2). To reconstruct, trace the LCS grid backwards: on a diagonal match, include that character once; otherwise include the character from whichever string you're stepping through (up or left).

How to drill subsequence DP

For each problem:
(1) say the brute force and its big-O;
(2) define dp[i] or dp[i][j] precisely in one sentence;
(3) write the recurrence as math before any code;
(4) identify the base cases (what does dp with a zero index mean?);
(5) state the time and space complexity;
(6) code it;
(7) dry-run on the edge cases — empty string(s), single character, all-same characters, two identical strings. If you can reconstruct the solution (not just its length), do it: trace back through the grid from the answer corner. That reconstruction step is a common follow-up and forces you to understand the table structurally, not just mechanically.

★ Review

Summary & self-test

Subsequence DP compresses an exponential search space into a polynomial table by asking a precise local question at each cell: "what is the best answer for this specific prefix (or pair of prefixes)?" Nail the definition of dp[i] or dp[i][j] first — the code follows almost mechanically.

02LIS recurrencedp[i] = max(dp[j]+1) for all j < i where a[j] < a[i]. Base: dp[i]=1. O(n²) time.
03O(n log n) LISTails array T: bisect_left to find position, replace or append. len(T) = LIS length. T is always sorted.
042-D: dp[i][j] = two prefixesFor LCS and Edit Distance, dp[i][j] aligns the first i chars of s1 with first j of s2. Base rows/columns all zero (or 0…n for Edit Distance).
05LCS recurrenceMatch → dp[i-1][j-1]+1 (diagonal). No match → max(dp[i-1][j], dp[i][j-1]) (skip a char from either string).
06Edit Distance recurrenceMatch → copy diagonal. No match → 1 + min(diagonal=replace, above=delete, left=insert).
07Counting vs optimizingOptimization uses max/min; counting uses addition. Same grid shape, different operator in the recurrence.
08Space reductionMost 2-D DP tables only need the previous row. Roll two rows for O(min(m,n)) space. Traverse carefully to avoid reading updated values.
Self-test before you move on

Cover the notes and answer aloud:
(1) For LIS, why is the answer max(dp) and not dp[n-1]? Give an example where they differ.
(2) In the O(n log n) LIS approach, why is it safe to replace an element of the tails array T — doesn't that corrupt the answer?
(3) Write the LCS recurrence from memory in two cases (match and no-match), and state what each of the three neighbors in the no-match case represents physically.
(4) In Edit Distance, what does dp[i][0] equal and why? What operation does "looking left" in the recurrence correspond to?
(5) How would you modify the LCS grid to instead count distinct subsequences? What operator changes and what do the base cases become?
(6) "Russian Doll Envelopes" — why do you sort heights descending for equal widths, and which algorithm do you run on the heights?
