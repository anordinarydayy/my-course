Bitmask DP

When your problem revolves around a small set of items — typically n ≤ 20 — and every distinct subset of those items is a meaningful state, you can encode that subset as a single integer and do dynamic programming over all 2n possible subsets. Bitmask DP turns exponential search into a tractable table fill.

00 The idea

What problem does it solve?

Consider this: you have 4 cities and you want to know the shortest route that visits all of them starting from city 0. How many routes are there? For n cities the number of permutations is n! — at n = 15 that is over a trillion. You cannot enumerate them all.

The insight is that you do not need to track the exact order of every city visited so far. You only need two things: which cities have been visited (the set), and where you currently are (so you know what transitions are available). If you store the visited set as a bitmask — an integer whose k-th bit is 1 when city k has been visited — then the total number of distinct states is 2n × n, which at n = 15 is about half a million. That is exactly the kind of compressed state space that dynamic programming can fill efficiently.

The pattern shows up whenever:

  • n ≤ 20 (often ≤ 15 in practice) and you need the optimal or feasible arrangement of all n items.
  • The state you care about is not the order of decisions but the subset chosen so far.
  • A naive permutation search gives O(n!) but the number of distinct subsets, 2n, is manageable.
The core bit operations

Four idioms appear in every bitmask DP: check if item k is in mask — (mask >> k) & 1; add item k — mask | (1 << k); remove item k — mask & ~(1 << k); full mask for n items — (1 << n) - 1. The last expression is the mask with bits 0 through n−1 all set.

01 Encoding subsets

A subset grows one bit at a time ▸

Before building the DP table, it is worth visualizing how a bitmask changes as items are added to the subset. Start from the empty mask (0), and for each new item k, set its bit with mask | (1 << k). The order in which bits are set determines the path through the 2n states, but the final mask for a given subset is always the same integer regardless of insertion order.

Below: four items labeled 0–3. Watch the binary representation update as items are toggled in, and notice how bit operations translate directly to set operations — adding item 2 flips exactly bit 2, leaving all other bits untouched.

Why integers, not sets?

A Python set cannot be a dictionary key (it is mutable). A frozenset can, but comparing and hashing it costs O(n) per lookup. An integer key hashes in O(1) and fits directly in a DP array indexed by mask. That is what makes bitmask DP fast in practice: the state table is just a list of lists, and every lookup is a constant-time array access.

Worked problemHardLeetCode 847
Shortest Path Visiting All Nodes

You are given an undirected graph of n nodes (n ≤ 12). Every edge has unit weight (cost 1 to traverse). Find the minimum number of edges you must cross to visit every node at least once. You may start at any node and revisit nodes and edges freely.

Think it through

Restate

I want the shortest walk (not path — revisits allowed) that covers every node. The answer is the minimum total edges crossed, minimized over all starting nodes.

Brute force

Try every permutation of nodes and for each compute the walking cost using BFS distances. That is O(n!) orderings times O(n² + edges) BFS per pair — completely infeasible for n = 12, which has nearly half a billion permutations. O(n! · n²).

The waste

The brute force tracks the exact order nodes were visited, but the problem only cares which nodes have been visited so far and where we currently stand. Two walks that reach the same node with the same visited-set are equivalent regardless of how they got there — we only need to keep the cheaper one.

The move — BFS on (node, mask) states

Model the state as a pair (current_node, visited_mask). Run BFS from every possible starting node simultaneously — each start node pushes state (node, 1 << node) with distance 0 into the queue. BFS guarantees the first time we reach a state with visited_mask == full_mask it is at minimum cost. At each step, for each neighbor v of the current node, expand to (v, mask | (1 << v)) if that state has not been seen.

Invariant & cost

BFS processes states level by level (each level = one extra edge). The state space is n × 2n. With n = 12 that is 12 × 4096 ≈ 49,000 states — tiny. Total: O(n · 2n) time, O(n · 2n) space.

The code

python

from collections import deque

def shortestPathLength(graph):
    n = len(graph)
    full = (1 << n) - 1          # all bits set = every node visited
    if n == 1: return 0

    # BFS queue: (node, visited_mask, distance)
    q = deque()
    seen = set()
    for start in range(n):
        state = (start, 1 << start)
        q.append((start, 1 << start, 0))
        seen.add(state)

    while q:
        node, mask, dist = q.popleft()
        if mask == full:
            return dist          # BFS → this is the minimum
        for nb in graph[node]:
            nxt = mask | (1 << nb)
            if (nb, nxt) not in seen:
                seen.add((nb, nxt))
                q.append((nb, nxt, dist + 1))
    return -1                      # unreachable (graph is connected, so never hit)
Narrate it to the interviewer

"I restate it as: find the minimum-length walk covering all nodes, starting anywhere, revisits allowed. The brute force is all permutations — O(n!) — which fails immediately at n = 12. The key observation is that the walk's history doesn't matter; only the current node and the visited set matter. So my state is a pair: current node and a bitmask of visited nodes. That gives me n × 2n states — about 49,000 for n = 12. I run BFS starting from every node at once, since BFS on a unit-cost graph always finds the shortest path first. Whenever I expand a neighbor v, I set bit v in the mask; the first time I hit a state where all n bits are set, that is my answer. Total: O(n · 2n) time and space."

Dry-run aloud: a single node (return 0 immediately), a disconnected graph (unreachable — but the problem guarantees connectivity, so this is just a guard). Two nodes with one edge — the BFS pushes both starting states; the state (0, mask=11) is reached at distance 1 from start node 0, which equals the full mask — done.

02 DP over subsets

dp[mask][i] — state at every subset ▸

The BFS approach from section 01 works for unit-edge graphs because BFS naturally finds the shortest path. For weighted graphs — where edges have different costs — we switch to dynamic programming. The DP table dp[mask][i] stores the minimum cost to have visited exactly the nodes in mask, ending at node i. We fill the table by iterating over all masks from smallest to largest, and for each mask we try adding one more unvisited node.

The recurrence is: for each mask, for each node i in mask, for each neighbor j not in mask, update dp[mask | (1<<j)][j] = min(dp[mask | (1<<j)][j], dp[mask][i] + cost[i][j]). This is the Held-Karp algorithm for the Travelling Salesman Problem (TSP), and it is the canonical bitmask DP.

Why iterate masks in ascending order?

When computing dp[mask][i], you look up dp[mask ^ (1 << i)][prev] — a mask that is strictly smaller (one fewer bit). If you fill from small to large, the values you need are always already computed. This is the topological order the DP requires: a subset is never processed before all its proper subsets.

03 TSP variant

Travelling salesman on a small graph

Shortest Path Visiting All Nodes (LeetCode 847, worked in section 01) allows revisits and has unit costs. The classical TSP goes further: given a complete weighted graph, find the minimum-cost tour that visits every node exactly once and returns to the start. Both problems use the same bitmask DP skeleton — the difference is the graph structure and whether a return edge is added at the end.

The TSP DP initialization differs slightly: instead of starting from all nodes simultaneously, we fix the start node (say node 0) and set dp[1 << 0][0] = 0 — only node 0 visited, currently at node 0, cost 0. All other entries start at infinity. Then the same recurrence fills the table. After the table is full, the answer is min over all i of dp[full_mask][i] + cost[i][0] — the cheapest way to be at any node with everyone visited, plus the return leg home.

BFS on (node, mask)

  • Works for unit-weight graphs — BFS level = edges crossed.
  • Can start from all nodes simultaneously (multi-source BFS).
  • No need to fill the full DP table — stops at first full-mask state.

Held-Karp DP table

  • Works for arbitrary weights — fills all 2n × n cells.
  • Fix a start node (often 0) to avoid redundant symmetric tours.
  • After filling, scan the last row and add return-edge costs.
Worked problemHardLeetCode 847
Shortest Path Visiting All Nodes (DP formulation)

Same as before — undirected graph, unit-weight edges, revisits allowed — but now let us write the DP table formulation rather than BFS, to see how Held-Karp maps directly to code.

Think it through

Restate

Fill a table dp[mask][i] = minimum edges crossed to have visited exactly the nodes in mask, currently at node i. Answer is min(dp[full][i]) over all i.

Brute force

All orderings of n nodes: O(n!) — ruled out immediately for n up to 12.

The waste

Two paths ending at the same node with the same visited set are interchangeable — only the cheaper one is worth keeping. The mask captures the visited set exactly.

The move

Initialize: for each start node i, set dp[1 << i][i] = 0 and all others to infinity. Then for each mask in order 1 to full_mask, for each node i in mask (bit i set), try each neighbor j of i and relax dp[mask | (1<<j)][j] via dp[mask][i] + 1.

Invariant & cost

dp[mask][i] always holds the true minimum when computed — subsets fill before supersets, so every dependency is satisfied. O(n² · 2n) time, O(n · 2n) space (the extra n factor comes from iterating neighbors for each mask-node pair).

The code

python

def shortestPathLengthDP(graph):
    n = len(graph)
    full = (1 << n) - 1
    INF = float('inf')

    # dp[mask][i] = min edges to visit exactly mask, ending at i
    dp = [[INF] * n for _ in range(1 << n)]

    # start from every node with just that node visited
    for i in range(n):
        dp[1 << i][i] = 0

    for mask in range(1, (1 << n)):
        for i in range(n):
            if not (mask >> i) & 1:   # i must be in mask
                continue
            if dp[mask][i] == INF:
                continue
            for j in graph[i]:       # try each neighbor j
                nxt = mask | (1 << j)
                dp[nxt][j] = min(dp[nxt][j], dp[mask][i] + 1)

    return min(dp[full])           # best ending node with all visited
Narrate it to the interviewer

"The state is which nodes have been visited — a bitmask — plus which node I am currently at. That gives me n × 2n states. I initialize by seeding every node as a valid starting point with cost 0, then fill masks in ascending order — smaller masks always before larger ones, which guarantees any value I look up is already computed. For each (mask, i) pair, I try to extend to each neighbor j, setting bit j in the new mask. The answer is the minimum over all ending nodes when the mask is full. This is O(n² · 2n) time. For n = 12 that is about 600,000 operations — well within a second."

Dry-run aloud: a single node (full = 1, dp[1][0] = 0, return 0). A path graph 0-1-2: dp[001][0]=0, dp[011][1]=1, dp[111][2]=2, answer 2.

04 Assignment / partition

Partition to k equal-sum subsets ▸

A related family of problems assigns items to groups or buckets rather than routing through nodes. The key insight is the same: the set of items already placed is the state. For Partition to K Equal Sum Subsets (LeetCode 698) you must decide whether an array of integers can be split into exactly k subsets each with the same sum. With n ≤ 16, the 2n subsets are manageable.

The DP state is a bitmask of which items have been assigned. dp[mask] is the running sum of the current bucket given that exactly the items in mask have been placed so far. Whenever the running sum hits the target, it resets to 0 for the next bucket. We build up from the empty mask to the full mask: for each mask, if dp[mask] ≥ 0 (reachable), try adding each item not yet in the mask; if adding it keeps the partial sum ≤ target, record the new partial sum in dp[mask | (1 << item)].

Worked problemMediumLeetCode 698
Partition to K Equal Sum Subsets

Given an array of positive integers and an integer k, decide whether the array can be partitioned into exactly k non-empty subsets with equal sum. Each element belongs to exactly one subset.

Think it through

Restate

Can I assign every element to one of k buckets so every bucket totals the same value, total / k? If total % k != 0, immediately return false.

Brute force

Recursively try assigning each element to any of k buckets and backtrack. With memoization on the bucket state it is still exponential without the bitmask insight. Pure backtracking is O(kn) without pruning.

The waste

Backtracking explores many equivalent orderings — filling bucket 1 with {4,2} and then bucket 2 with {3,3} is the same as the reverse order. The bitmask representation of which items have been placed collapses all these orderings into a single state.

The move

Let dp[mask] = the partial sum of the currently filling bucket after placing exactly the items in mask, or -1 if mask is unreachable. Initialize dp[0] = 0. For each reachable mask, try adding each item not yet placed: if dp[mask] + nums[i] ≤ target, the new partial sum is (dp[mask] + nums[i]) % target — it wraps to 0 when a bucket completes. Set dp[mask | (1<<i)] to this value if not already set.

Invariant & cost

dp[mask] ≥ 0 means this assignment of items is valid so far. The answer is dp[full_mask] == 0 — all items placed, current bucket's partial sum is 0 (meaning the last bucket completed exactly). O(n · 2n) time, O(2n) space.

The code

python

def canPartitionKSubsets(nums, k):
    total = sum(nums)
    if total % k != 0:
        return False
    target = total // k
    n = len(nums)
    if max(nums) > target:          # any single element exceeds quota → impossible
        return False

    # dp[mask] = partial sum of the current bucket, -1 if unreachable
    dp = [-1] * (1 << n)
    dp[0] = 0                        # empty mask: no items placed, sum = 0

    for mask in range(1 << n):
        if dp[mask] == -1:
            continue                 # unreachable state, skip
        for i in range(n):
            if (mask >> i) & 1:
                continue             # item i already placed
            nxt = mask | (1 << i)
            if dp[nxt] != -1:
                continue             # already computed a valid path
            candidate = dp[mask] + nums[i]
            if candidate <= target:
                dp[nxt] = candidate % target  # wraps to 0 when bucket fills

    return dp[(1 << n) - 1] == 0    # all items placed, last bucket complete
Narrate it to the interviewer

"I first check if the total is divisible by k and if any single element exceeds the target quota — both are early exits. Then I define dp[mask] as the partial sum of whichever bucket is currently being filled, given that the items in mask have already been assigned. I initialize dp[0] = 0 and set everything else to -1 (unreachable). For each reachable state, I try adding each unplaced item; if the candidate sum stays within the target, I record candidate % target in the next state — this automatically resets to 0 when a bucket is completed, so the same logic handles all k buckets without a separate counter. The answer is whether the full mask ends up with partial sum 0. Time: O(n · 2n), space: O(2n)."

Dry-run aloud: nums=[2,2,2,2], k=2 — target=4, dp progresses cleanly; answer true. nums=[1,2,3,4], k=3 — target cannot be formed (sum=10, 10%3≠0), return false immediately. Single element equal to target: dp[0]=0, add it to get dp[1]=0, valid.

05 Submasks & cost

Iterating submasks and the real complexity

Sometimes you need to iterate over all subsets of a given mask — for instance, in problems that assign a group of items (a submask) to one worker and handle the rest separately. The standard loop is:

# iterate all non-empty submasks of `mask` in decreasing order
sub = mask
while sub > 0:
    # process sub as a subset of mask
    sub = (sub - 1) & mask  # clever bit trick: peel off lowest set bit, keeping only bits in mask

The expression (sub - 1) & mask works because subtracting 1 flips the lowest set bit and all lower bits, and ANDing with mask throws away any bit not in the original mask — giving the next smaller submask.

The true cost of submask enumeration

If you iterate all submasks of every mask, the total iterations are 3n, not 4n. The proof: each of the n bits independently contributes to three cases — in the outer mask but not the submask, in both, or in neither. This is a classic result and interviewers often ask you to justify it.

For the main bitmask DP loop (no submask iteration), the cost is O(2n · n) — for each of 2n masks you loop over n items. For n = 20 that is about 20 million operations — comfortably within a second. For n = 20 with submask iteration: 320 ≈ 3.5 billion — too slow, so submask DP is typically limited to n ≤ 15.

Recognizing bitmask DP in an interview

Three signals:
(1) n ≤ 20 and the problem asks for an optimal arrangement or covering of all n items;
(2) two states of each item — in/out, assigned/unassigned, visited/unvisited;
(3) the phrase "subset" or "partition" in the problem. The three complexity tiers to quote: O(n · 2n) for table fill, O(n² · 2n) when relaxing edges per cell, O(3n) for submask enumeration.

What the interviewer asks next

"Can you prove the 3n submask bound?" — for each of the n bit positions, a bit is either (a) not in mask at all, (b) in mask and in sub, or (c) in mask but not in sub. Three choices per bit, n bits: 3n. "What if n is 30?" — bitmask DP becomes infeasible (230 ≈ 10⁹ states), and you need a different approach: meet-in-the-middle splits the items into two halves of size 15, solves each independently, and joins — O(2n/2 · n). "Why not just use recursion with memoization?" — you can, and it is equivalent; the iterative table is faster in practice because it avoids Python function call overhead and guarantees the topological order explicitly.

Idea Classic Held-Karp: dp[mask][i] = min cost of visiting exactly the cities in mask, ending at i; answer is min over i of dp[full][i] + cost[i][0]. O(n² · 2n).

Idea Enumerate all submasks of the order mask; dp[mask] = true if orders in mask can be fulfilled by the first popcount(mask) item types. Requires 3m submask iteration — keep m ≤ 15.

Idea dp[mask] = remaining time in the current session after scheduling tasks in mask. When adding a task would overflow, start a new session. O(n · 2n).

An array of 2n integers; in each of n operations pick a pair and earn gcd(a, b) × op_number. Maximize total score.

Idea dp[mask] = max score using the items in mask; the operation number is popcount(mask)/2. Try all pairs (i,j) in the complement mask. O(n² · 2n).

How to drill bitmask DP problems

Before touching the keyboard:
(1) confirm n ≤ 20 and that the state is a subset;
(2) write dp[mask] or dp[mask][i] in plain English — "minimum cost to visit exactly the cities in mask, currently at city i";
(3) write the recurrence as a one-liner;
(4) identify the base case (usually mask = 0 or single-bit masks). Then code, test on the minimum example (n = 2 or 3), and dry-run the edge cases: all items equal, k = 1 (entire array is one subset), n = 1. State the big-O out loud — naming the two factors 2n and n separately shows you understand the structure.

★ Review

Summary & self-test

Bitmask DP compresses an exponential search space by observing that the relevant state is which subset of items has been processed, not the exact order. When n ≤ 20, the 2n possible subsets are small enough to enumerate, and each subset is stored as a single integer. The key ingredients are: the right DP state definition, filling masks in ascending order (smaller subsets before larger), and knowing the four bit operations by heart.
