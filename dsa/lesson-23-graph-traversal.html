Graph Traversal — BFS & DFS

Grids, networks, maps, dependency chains — almost any problem that asks "how many connected groups?" or "what is the shortest path?" is secretly a graph traversal problem in disguise. BFS and DFS are the two engines that power this entire family, and knowing when to reach for each one is the core skill of graph-heavy interviews.

Original study notes. Written from scratch for coding-interview preparation, covering graph traversal patterns common in SWE interviews across tech companies. Problem names below are used only as well-known identifiers (Number of Islands, Flood Fill, Rotting Oranges, Clone Graph); every prompt, diagram, and line of code here is my own. Contains three interactive diagrams (press Play or step through with Next) and three fully worked problems with reasoning chains, code, and interviewer narration scripts.
00 The model

Thinking in graphs

A graph is just a set of nodes (things) connected by edges (relationships). That definition is broad enough to cover almost anything: cities linked by roads, web pages linked by hyperlinks, variables linked by dependencies — and most commonly in interview problems, grid cells linked to their four neighbors.

When you see a 2-D grid, the translation is mechanical: every non-wall cell is a node; two cells are connected if they are adjacent horizontally or vertically (and not separated by a wall or water). This lets you apply every graph algorithm you know without writing any special grid logic — you just define a "get neighbors" helper that returns the up-to-four adjacent cells within bounds.

Two characteristics of any traversal problem you should name up front:

  • V + E — every correct BFS or DFS visits each node (V) and edge (E) at most once, for O(V + E) time. On an m × n grid that becomes O(m · n) since each cell has at most 4 neighbors.
  • Visited set — without it, the algorithm can revisit cells forever (or loop in a cycle). The first thing you add to any graph template is a way to mark "already seen."
The choice in one sentence

Reach for DFS when you need to know whether a path exists, count connected components, or mark an entire region. Reach for BFS when you need the shortest (fewest-hops) path in an unweighted graph — BFS visits nodes in non-decreasing order of distance from the source, so the first time it reaches a destination is the shortest way there.

01 DFS — depth first

DFS for reachability and connected components

Depth-first search commits fully to one direction before backtracking. Starting at a node, it marks the node visited and immediately recurses into any unvisited neighbor — pushing deeper and deeper until it hits a dead end, then unwinding. The call stack is the frontier; you never need an explicit queue.

The brute-force answer to "count connected land masses in a grid" is to start a counter at zero, then scan every cell; when you find an unvisited land cell, increment the counter and visit its entire cluster. Without DFS, you'd need to re-scan neighbors repeatedly — potentially touching each cell many times. DFS fixes this: once a cell is marked, it is never entered again, so the total work is O(m · n) across the whole grid.

Below, watch DFS explore a small 4×4 binary grid. Each step picks the next unvisited land cell reachable from the current one, marks it, and recurses. When the full island is painted, the DFS returns and the outer loop continues scanning for the next unvisited land cell to start a new component.

DFS (recursive / stack)

  • O(m·n) time and space (call stack depth = island size).
  • Natural fit for connected-components, cycle detection, topological sort.
  • Memory use is proportional to the deepest path — can be large on a fully connected grid.

DFS (iterative with explicit stack)

  • Same O(m·n) complexity, avoids Python's recursion limit.
  • Use stack = [start]; while stack: curr = stack.pop().
  • Preferred in interviews when grid is very large or recursion depth is a concern.
Worked problemMediumLeetCode 200
Number of Islands

Given an m × n grid of '1' (land) and '0' (water) characters, count how many distinct connected land masses exist. Two land cells are part of the same island if they are adjacent horizontally or vertically.

Think it through

Restate

I need to count connected components in a binary grid where edges connect horizontally or vertically adjacent land cells.

Brute force

For every land cell, repeatedly scan all other cells to find which are reachable — effectively a nested scan. That approaches O((m·n)²), and on a 300×300 grid that is way too slow.

The waste

The nested scan re-examines cells we've already assigned to a component. Once a cell is visited and marked, we never need to look at it again — but the brute force does not record that.

The move

Scan every cell with two nested loops. When I land on an unvisited '1', increment the island count and run DFS from that cell to mark the entire connected component as visited. The outer loop then continues — any cell already marked is skipped instantly.

Invariant & cost

Every cell is entered by DFS at most once (it is marked on entry). The outer loop touches every cell once. So total work is O(m·n) time, O(m·n) space for the visited state (we mutate the grid in-place to avoid extra memory).

The code

python

def numIslands(grid):
    if not grid: return 0
    rows, cols = len(grid), len(grid[0])
    count = 0

    def dfs(r, c):
        # out of bounds or water or already visited: stop
        if r < 0 or r >= rows or c < 0 or c >= cols or grid[r][c] != '1':
            return
        grid[r][c] = '2'   # mark visited (in-place)
        dfs(r - 1, c); dfs(r + 1, c)
        dfs(r, c - 1); dfs(r, c + 1)

    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == '1':
                dfs(r, c)
                count += 1
    return count
Narrate it to the interviewer

"The problem is counting connected components in a binary grid. The naive approach re-examines every cell for every island — quadratic. Instead I scan every cell once; when I hit an unvisited '1' I start a DFS that marks the whole connected land mass as '2' (visited), then I increment my island counter. Since each cell is entered by DFS at most once and the outer scan also touches each cell once, the total is O(m·n) time. I'm mutating the grid in-place so I need no extra visited set — O(m·n) implicit stack space in the worst case (one giant island), but no additional array."

Dry-run aloud: an all-water grid (0 islands), a single land cell (1 island), a fully connected land grid (1 island), and a grid where every cell alternates (m·n/2 islands).

What the interviewer asks next

"What if you can't mutate the input?" → use a separate visited set of tuples (r, c), same complexity. "Recursion limit concern?" → convert to an iterative DFS with an explicit stack. "What about diagonal adjacency?" → add four more directions to the neighbor list. "What is V and E here?" → V = m·n cells, E ≤ 4·m·n edges (each cell has at most 4 neighbors), so O(V + E) = O(m·n).

02 BFS — breadth first

BFS for shortest unweighted distance

Breadth-first search expands outward in concentric rings from the source. It processes every cell at distance d before it processes any cell at distance d+1. This layering property is the key insight: when BFS first reaches a cell, it has taken the fewest possible hops to get there. No shorter path exists, because if one did, BFS would have found the cell in an earlier ring.

DFS, by contrast, might reach a destination via a very long path before a short one — it has no guarantee about hop-count ordering. So the rule is simple: if the question asks for the minimum number of steps or the shortest distance in an unweighted graph, reach for BFS.

The mechanics: seed a queue with the starting node (mark it visited), then loop — dequeue a node, enqueue all its unvisited neighbors (mark them visited on enqueue, not dequeue), and track how many complete layers you have processed. Each full layer is one more hop from the source.

Why BFS is correct for shortest paths

The queue enforces FIFO (first-in, first-out) order. Nodes at distance d are all enqueued before any node at distance d+1 (because they are enqueued by processing distance d-1 nodes, which come before d nodes in the queue). So when you dequeue a node for the first time, no shorter path can still be in the queue — it would already have been processed. This is the invariant: the first time BFS reaches a node is via the shortest path to it.

Worked problemEasyLeetCode 733
Flood Fill

An image is an m × n grid of integers representing pixel colors. Starting from a given pixel (sr, sc), repaint that pixel and all pixels reachable from it (via 4-directional adjacency) that share the original color of the starting pixel, all to a new color. Return the modified grid.

Think it through

Restate

From a starting pixel, find every pixel in its connected same-color region and repaint them all. This is a single connected-component traversal — DFS or BFS, my choice.

Brute force

For each pixel in the grid, check whether it is reachable from the start and has the matching color, then repaint. That means running a BFS/DFS for every pixel to check reachability: O((m·n)²).

The waste

I only need one traversal starting from (sr, sc). Any pixel the traversal reaches is by definition connected and same-color — I don't need a separate reachability check per pixel.

The move

Save the old_color. If old_color == new_color, return immediately (already done, would also cause infinite recursion). Otherwise run DFS/BFS from (sr, sc): at each cell, if it is in-bounds and still shows old_color, repaint it to new_color and recurse into its four neighbors. The repaint itself serves as the visited marker.

Invariant & cost

Each cell is repainted at most once (after which it no longer matches old_color and will not be entered again). O(m·n) time, O(m·n) space (stack depth in the worst case).

The code

python

def floodFill(image, sr, sc, color):
    old = image[sr][sc]
    if old == color: return image   # nothing to do
    rows, cols = len(image), len(image[0])

    def dfs(r, c):
        if r < 0 or r >= rows or c < 0 or c >= cols:
            return
        if image[r][c] != old: return  # wrong color or already painted
        image[r][c] = color              # paint = mark visited
        dfs(r - 1, c); dfs(r + 1, c)
        dfs(r, c - 1); dfs(r, c + 1)

    dfs(sr, sc)
    return image
Narrate it to the interviewer

"This is a single-component DFS/BFS from a starting pixel. The naive path is to check reachability for every pixel independently — quadratic. Instead I run one DFS from the start pixel. I save the original color, guard against the case where old equals new (which would create an infinite loop since the repainted cell still looks like old), then at each step: if the cell is in-bounds and still holds the old color, repaint it to the new color (which is simultaneously the visited marker) and recurse into four neighbors. Every cell is repainted at most once, so the total is O(m·n) time and O(m·n) stack space."

Dry-run aloud: old_color == new_color (early exit), a 1×1 grid (no neighbors to explore), and a grid where only the starting cell matches (no propagation).

03 Visited set

Counting components and avoiding revisits

Both DFS and BFS share the same critical contract: mark a node visited before (or immediately upon) entering it, and never enter a visited node again. Getting this wrong is the single most common graph bug — and it costs candidates the round.

There are two natural ways to mark visited in a grid problem. First: mutate the grid in-place (e.g., set a '1' to '2' or to '0'). This avoids any extra memory and is usually fine when the input is a local copy. Second: keep an explicit visited set of tuples (r, c) — clean and safe when you must not modify the input. Both give the same O(m·n) time; the set adds O(m·n) extra space.

A subtle timing rule: for BFS, mark a cell visited when you enqueue it, not when you dequeue it. If you wait until dequeue, the same cell can be added to the queue multiple times before you get to process it — wasting work and possibly producing wrong counts.

The reusable shape

The outer loop of "count connected components" is the same in every graph: iterate over all nodes; when you find an unvisited one, increment your component counter and run a full DFS/BFS from it. This shape works for Number of Islands, Number of Provinces, Friend Groups — they are all the same pattern with different "nodes" and "edges."

04 Multi-source BFS

Seeding multiple sources at distance zero

Single-source BFS starts with one node at distance 0. But some problems have multiple starting points, all equally close to the goal at the start. The naive fix is to run a separate BFS from each source and take the minimum — but that is O(sources × m × n) and collapses for large inputs.

The key insight: just seed the queue with all sources at the same time, all at distance 0. BFS then expands outward from all of them simultaneously, exactly as if there were one giant super-source connected to each real source. Each cell is reached by the closest source, in the same total O(m·n) work as a single-source BFS.

Rotting Oranges (LeetCode 994) is the canonical example: rotten oranges are the sources; fresh oranges are the cells to be reached; each minute is one BFS level. The answer is the number of BFS levels needed to infect all fresh oranges — or -1 if some fresh orange is isolated.

Worked problemMediumLeetCode 994
Rotting Oranges

A grid contains empty cells (0), fresh oranges (1), and rotten oranges (2). Each minute, every fresh orange that is directly adjacent to a rotten one also becomes rotten. Return the minimum number of minutes until all fresh oranges are rotten, or -1 if it is impossible.

Think it through

Restate

Spread rot from all initially rotten oranges simultaneously, one hop per minute, and count how many minutes until no fresh oranges remain. If a fresh orange is unreachable, return -1.

Brute force

Simulate minute by minute: scan the entire grid each minute looking for fresh oranges adjacent to rotten ones, infect them, repeat until stable. That's O(minutes × m × n) — in the worst case, O((m·n)²) if infection spreads only one cell per minute across the whole grid.

The waste

The simulation rescans the whole grid each minute, but only cells adjacent to the current infection frontier can change. BFS naturally processes only the frontier — we just need to track it efficiently.

The move — phase 1

Scan the grid once: add all initially rotten cells to the BFS queue at level 0, and count all fresh oranges. If there are no fresh oranges, return 0 immediately.

The move — phase 2

Run multi-source BFS. Each time you complete a full level of the queue (a "minute"), increment the timer. When you infect a fresh orange, decrement the fresh count. After BFS finishes, if the fresh count is 0 return the timer; otherwise return -1 (isolated fresh oranges remain).

Invariant & cost

Each cell is enqueued at most once. Total work is O(m·n) time, O(m·n) space for the queue.

The code

python

from collections import deque

def orangesRotting(grid):
    rows, cols = len(grid), len(grid[0])
    queue = deque()
    fresh = 0

    # phase 1: seed all rotten oranges; count fresh
    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == 2: queue.append((r, c))
            elif grid[r][c] == 1: fresh += 1

    if fresh == 0: return 0  # already done
    minutes = 0
    dirs = [(-1,0),(1,0),(0,-1),(0,1)]

    # phase 2: multi-source BFS level by level
    while queue and fresh > 0:
        minutes += 1
        for _ in range(len(queue)):  # process one full level
            r, c = queue.popleft()
            for dr, dc in dirs:
                nr, nc = r + dr, c + dc
                if 0 <= nr < rows and 0 <= nc < cols and grid[nr][nc] == 1:
                    grid[nr][nc] = 2
                    fresh -= 1
                    queue.append((nr, nc))

    return minutes if fresh == 0 else -1
Narrate it to the interviewer

"The question is minimum minutes until all fresh oranges rot — that's 'minimum distance' language, which points to BFS. But there are multiple rotten sources, all equally at the start. The key move is multi-source BFS: I seed the queue with every initially rotten orange at level 0, count the fresh ones, then expand level by level — each level is one minute. Whenever I infect a fresh orange, I decrement the fresh counter and enqueue it. After BFS I check whether fresh is zero. If not, some oranges are walled off — return -1. The whole thing is O(m·n) time: each cell enters the queue at most once."

Dry-run aloud: no fresh oranges at all (return 0 before BFS starts), a grid with one isolated fresh orange surrounded by empty cells (return -1), and a grid where all oranges start rotten (return 0).

What the interviewer asks next

"Why can't you just BFS from each rotten orange separately and take the minimum per cell?" → the max of those minima is the answer, not the minimum — and that can double-count correctly reachable cells. Multi-source BFS gives the right answer in one pass. "What if the grid has walls?" → walls are just cells you skip (grid[r][c] == 0 or 'wall'), same mechanics. "What if infection needs 2 steps instead of 1 per minute?" → change the step size in your neighbor loop; BFS still handles it. "What is the worst-case input?" → a zigzag path of fresh oranges that must be infected one by one.

05 Recognition

Spotting graph traversal, and the traps

Graph traversal is a large pattern — you need to recognize several different problem shapes and map them to the right traversal.

  • Count connected groups — any "how many clusters/islands/provinces/friend circles?" question. DFS or BFS from each unvisited node, increment a counter. Examples: Number of Islands, Number of Provinces.
  • Minimum hops / shortest path (unweighted) — "minimum steps", "fastest way", "fewest moves". BFS. Examples: 01 Matrix, Shortest Path in Binary Matrix.
  • Flood fill / paint a region — DFS from a starting node, repaint everything reachable. Paint the repainted color serves as visited marker. Example: Flood Fill.
  • Multi-source spread — "spread from all of X simultaneously". Seed BFS with all sources. Examples: Rotting Oranges, 01 Matrix (all 0s as sources).
  • Cycle detection — DFS with a "currently on the call stack" flag (for directed graphs) or a union-find structure (for undirected).
  • Topological order — dependencies must be processed before dependents. DFS post-order reversal, or Kahn's algorithm (BFS with in-degree counts).

Common bugs that cost candidates the round:

  • Marking visited on dequeue instead of enqueue (BFS). The same cell gets added to the queue multiple times. Mark it the moment you push it, not when you pop it.
  • Forgetting the out-of-bounds check before accessing grid[r][c]. Python will not crash immediately (it'll just IndexError), but it signals a gap in the template.
  • Using DFS where BFS is needed. DFS does not guarantee shortest paths — it just guarantees it reaches every reachable node. The moment a question mentions "minimum steps" or "shortest", switch to BFS.
  • Not counting fresh oranges before BFS. If you only check the queue after BFS, a freshly isolated orange (never enqueued, never processed) will be missed and you'll return the wrong answer.
Decision tree

Does the problem ask for reachability or counting? — DFS. Does it ask for minimum hops or distance? — BFS. Are there multiple equal-priority starting points? — multi-source BFS (seed all sources at level 0). Is the graph directed and you need a processing order? — topological sort (DFS or Kahn's). Is the graph weighted (edges have different costs)? — neither plain BFS nor DFS; use Dijkstra's (see later lessons).

EasyFlood FillDFS / flood

Starting from one pixel in an image grid, repaint its entire same-color connected region with a new color.

Idea Single DFS from the start; repaint serves as the visited marker. O(m·n) time.

MediumNumber of IslandsDFS / components

Count how many distinct groups of adjacent '1' cells exist in a binary grid of '1's and '0's.

Idea Outer loop over all cells; when you hit an unvisited '1', DFS to sink the island and increment count. O(m·n).

MediumClone GraphDFS / graph copy

Given a connected undirected graph node, produce a deep copy of the entire graph. Nodes hold integer values.

Idea DFS with a cloned map from original node to its copy; when you visit a neighbor, create its clone if not already in the map, then recurse. O(V + E).

MediumNumber of ProvincesDFS / union-find

You are given an n×n adjacency matrix where entry (i,j) is 1 if city i and city j are directly connected. Count how many disconnected city groups exist.

Idea Same "count components" pattern as islands, but the graph is given as an adjacency matrix instead of a grid. DFS from each unvisited city. O(n²).

MediumRotting Orangesmulti-source BFS

All rotten oranges spread infection to fresh neighbors each minute. Find the minimum minutes until all are rotten, or -1 if impossible.

Idea Seed BFS queue with every rotten orange; expand level by level; count levels and fresh oranges. O(m·n).

Medium01 Matrixmulti-source BFS

Given a binary matrix, find for each cell its distance to the nearest '0' cell (distance = number of steps).

Idea Multi-source BFS from all '0's simultaneously — same pattern as Rotting Oranges but all sources at the start. Each '1' cell gets its distance when first reached. O(m·n).

MediumSurrounded RegionsDFS from boundary

In a grid of 'X' and 'O', flip all 'O' regions that are completely surrounded by 'X', leaving the rest intact.

Idea Any 'O' touching the border is safe. DFS from every border 'O' to mark the safe region, then flip everything else. O(m·n).

MediumMax Area of IslandDFS / count cells

Find the island (connected group of '1's) with the largest cell count and return that count.

Idea DFS returns the size of the island it sinks; track the maximum across all starting DFS calls. O(m·n).

MediumCourse Schedulecycle detection / topo sort

Given n courses and prerequisite pairs, determine whether it is possible to finish all courses (i.e., no circular prerequisites).

Idea Model prerequisites as a directed graph; run DFS with a "currently on stack" flag to detect back-edges (cycles). O(V + E).

HardShortest Path in Binary MatrixBFS / 8-directional

Find the length of the shortest path from the top-left to the bottom-right of a binary matrix, moving in any of 8 directions through '0' cells. Return -1 if no path exists.

Idea BFS from (0,0) if it is '0'; expand in 8 directions. The first time you reach (n-1,n-1) is the shortest path. O(n²).

HardWord LadderBFS / implicit graph

Transform a begin-word into an end-word by changing one letter at a time, where each intermediate word must exist in a given word list. Return the minimum number of transformations.

Idea Implicit graph: each word is a node; edges connect words that differ by one letter. BFS from begin-word; each level is one transformation. O(n · L²) where L is word length.

HardPacific Atlantic Water Flowreverse multi-source BFS

Water flows from a cell to an adjacent cell only if the neighbor's height is ≤ the current cell's. Find all cells from which water can flow to both the Pacific and the Atlantic.

Idea Run BFS backwards from each ocean's boundary inward (climbing uphill). Cells reachable from both BFS sets are the answer. O(m·n).

How to drill graph traversal problems

For each problem:
(1) name the graph model — what are V and E?;
(2) choose DFS or BFS and explain why;
(3) decide the visited strategy (mutate vs. set);
(4) identify whether it's single-source, multi-source, or component counting;
(5) state the complexity as O(V + E);
(6) code the template;
(7) dry-run at least three edge cases: an empty grid, a single cell, and a grid with no valid path. Narrating all six steps before touching the keyboard is what separates strong graph candidates from weak ones.

★ Review

Summary & self-test

Graph traversal is a single family of algorithms under a shared template: visit nodes, mark them as you go, never enter a visited node twice. The whole skill is knowing which traversal to run (DFS or BFS), where to seed it (one source, many sources, or all unvisited nodes), and what to compute as it runs (component count, distance, region size).

01Grids are graphsEvery non-wall cell is a node; 4-directional adjacency provides the edges. O(V+E) = O(m·n).
02DFS for componentsRecurse (or use a stack) to sink entire connected regions. Good for counting, cycle detection, reachability.
03BFS for shortest hopsQueue-based; expands ring by ring. First arrival = shortest path in unweighted graphs.
04Visited set is mandatoryMark on enqueue (BFS) or on entry (DFS). Skipping this creates infinite loops and wrong counts.
05Multi-source BFSSeed the queue with all sources at distance 0. Same cost as single-source — one O(m·n) pass.
06The templateOuter loop over nodes; when unvisited, trigger traversal and do work (count, fill, measure). Inside: check bounds, check visited, mark, enqueue/recurse.
07Mark on enqueue, not dequeueBFS-specific: marking on dequeue allows duplicates in the queue and breaks level-counting.
08ComplexityBoth DFS and BFS: O(V + E) time, O(V) space. On a grid: O(m·n) for both.
