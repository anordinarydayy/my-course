Cyclic Sort / Index as Hash

When an array holds integers drawn from a small, known range, the array indices themselves can serve as a hash table — placing value v at index v − 1 by swapping gives you a sorted arrangement in O(n) time and O(1) extra space. That single pass unlocks a whole family of missing-number and duplicate-detection problems that would otherwise need a set or a sort.

00 The idea

The array is the hash table

Imagine you are handed the array [3, 1, 4, 2] and told every value is in the range 1 to 4 — no value is larger than the array's length. A conventional solution would sort it (O(n log n)) or build a hash set (O(n) time but O(n) extra space). But there is a cheaper observation: value 1 belongs at index 0, value 2 belongs at index 1, and so on. The array can hold them all without any extra data structure — it just needs rearranging.

The rearranging trick is to walk index i from left to right, and at every position ask: "Is the value here already in its home slot?" If not, swap it with whatever sits at its correct index. Repeat until either the value is home or the home slot already holds the right value (meaning we hit a duplicate and must move on). After one pass every value that can go home has gone home. A second O(n) pass then reads off whatever anomaly the problem asks for — a missing value, all disappeared values, a repeated value — simply by scanning for positions where a[i] ≠ i + 1.

The reusable shape

Cyclic sort applies whenever: (1) the values are integers in a contiguous range (1..n or 0..n−1), and (2) you want O(1) extra space. The two-pass structure is always the same — one cyclic-sort pass, then one scan for the answer. Only the range formula and the scan condition change between problems.

01 Cyclic placement

Sending every value home

The brute force for sorting an arbitrary array is comparison sort at O(n log n). But when values are constrained to 1..n — a fact most problems state up front — we can exploit the direct mapping value v → index v − 1 and sort in O(n) time with O(1) extra space.

Walk a cursor i from left to right. At each position let v = a[i] and compute home = v − 1. If i ≠ home and a[home] ≠ v (the home slot does not already hold the right value), swap a[i] with a[home] and do not advance i — the new a[i] might also need swapping. If the swap condition fails (either we are already home, or the home slot holds a duplicate), advance i. Each swap places at least one value permanently in its home, so the total number of swaps is at most n. Combined with the n steps of the outer cursor, the algorithm is O(n) time, O(1) space.

Why it is still O(n)

The inner while-loop might seem to make this O(n²), but each swap permanently sends a value to its home. A value can only be swapped home once, so across the entire algorithm there are at most n swaps. Total work: n (cursor steps) + n (swaps) = O(n).

Worked problemEasyLeetCode 268
Missing Number

You are given an array of n distinct integers, each in the range 0 to n (so there are n + 1 possible values but only n slots). Exactly one number in 0..n is absent. Return it using O(1) extra space.

Think it through

Restate

I have n distinct values from 0..n in an n-element array. One value is missing. Return it without allocating extra space.

Brute force

Sort the array (O(n log n)) and scan for the first gap, or store values in a hash set and check 0..n for the absent one — O(n) time but O(n) space.

The waste

The hash set re-encodes information already implicit in the array itself. If the values were 1..n I could cyclic-sort and look for the gap. The range 0..n is almost the same — I just adapt the home formula: value v goes to index v (not v − 1), since 0 now belongs at index 0.

The move — phase 1

Cyclic-sort the array: for each position i, while a[i] < n (n cannot go anywhere; there is no index n) and a[i] ≠ i, swap a[i] with a[a[i]]. After sorting, values 0..n−1 sit at their home indices wherever possible.

The move — phase 2

Scan left to right: the first index where a[i] ≠ i reveals the missing number — it is i itself. If all indices 0..n−1 are settled, then n is missing (it has no home slot), so return n.

Invariant & cost

After phase 1 every value that has a home slot sits there. Phase 2 is a single linear scan. Total: O(n) time, O(1) space.

The code

python

def missingNumber(nums):
    n = len(nums)
    i = 0
    while i < n:
        home = nums[i]              # value v lives at index v (range 0..n)
        if home < n and nums[home] != nums[i]:
            nums[i], nums[home] = nums[home], nums[i]   # send home
        else:
            i += 1                 # already home or n has no slot
    for i in range(n):
        if nums[i] != i:           # gap found
            return i
    return n                       # all 0..n-1 settled; n is missing
Narrate it to the interviewer

"The array has n elements from 0..n, one absent. The naive fix is a hash set — O(n) space — but since values map directly to indices I can use the array as the hash table. I cyclic-sort: for each position, while the value sitting there has a home elsewhere and that home is free, swap it in. After sorting, the first index i where a[i] ≠ i is the missing number; if all are settled, n itself is missing. Two O(n) passes — time O(n), space O(1)."

Dry-run aloud: [0, 1, 2] with n = 3 (missing is 3); [1, 0] (missing is 2); [0] (missing is 1). Confirm the n-has-no-slot guard so we never index out of bounds.

What the interviewer asks next

"Can you solve it without sorting?" — Yes: use math. The expected sum of 0..n is n*(n+1)/2; the actual sum is what we read. The difference is the missing value. One pass, O(1) space, no mutation. Mention it as the cleaner alternative, but note that cyclic sort generalises to harder variants (multiple missing values) where the math trick fails.

02 Extending the scan

Reading out every missing value

The single-missing-number problem has a clean math shortcut (sum formula). But what if multiple values are absent and you need all of them? The math trick does not extend — you would need to know the sum of the squares or use multiple equations, which gets messy. The cyclic sort approach, however, extends trivially: after the sort pass, every index i where a[i] ≠ i + 1 is a vacancy, meaning value i + 1 is missing. Collect all such indices in a single scan.

The brute force is to load the values into a hash set and then check each of 1..n for membership — O(n) time, O(n) space. The cyclic-sort approach cuts the space to O(1) (not counting the output list, which can be as large as n in the worst case and cannot be avoided). The key insight is that after sorting, the array is a perfect 1-to-1 mapping between indices and expected values, so finding all gaps is a single O(n) read.

Worked problemEasyLeetCode 448
Find All Numbers Disappeared in an Array

An n-element array contains integers each in 1..n, but some values appear more than once and others are absent. Return a list of every value in 1..n that is not present, using O(1) extra space (ignoring the output list).

Think it through

Restate

Values are from 1..n, with possible duplicates and absences. Find all missing values in one linear pass after sorting the array in place.

Brute force

Collect values into a hash set, then iterate 1..n and record what's not in the set. O(n) time, O(n) space.

The waste

The hash set duplicates information the array already encodes. Values 1..n map cleanly to indices 0..n−1. After a cyclic sort pass, any slot where a[i] ≠ i + 1 marks a vacancy — value i + 1 is missing.

The move — phase 1

Cyclic sort with range 1..n: for each i, let home = a[i] − 1. While a[i] ≠ a[home] (home holds a different value — not a duplicate), swap a[i] with a[home]. If we hit a duplicate, just advance i.

The move — phase 2

Walk indices 0..n−1. Wherever a[i] ≠ i + 1, append i + 1 to the result list.

Invariant & cost

Every value that can go home is at its home index after phase 1. Phase 2 reads the gaps. O(n) time, O(1) auxiliary space.

The code

python

def findDisappearedNumbers(nums):
    i = 0
    while i < len(nums):
        home = nums[i] - 1         # value v belongs at index v-1
        if nums[home] != nums[i]:  # home holds a different value: swap
            nums[i], nums[home] = nums[home], nums[i]
        else:
            i += 1                 # already home or duplicate: advance
    result = []
    for i in range(len(nums)):
        if nums[i] != i + 1:      # gap: value i+1 is missing
            result.append(i + 1)
    return result
Narrate it to the interviewer

"Values are in 1..n so each one has a home index at value − 1. I cyclic-sort in place: at each position I look at the home slot; if it holds a different value I swap, otherwise I advance. After the pass, every index where a[i] ≠ i + 1 is a vacancy — I collect i + 1 in a list. Two O(n) sweeps, O(1) auxiliary space."

Dry-run aloud: [4, 3, 2, 7, 8, 2, 3, 1] (n=8, missing are 5 and 6); all values present (empty result); all values identical (n−1 values missing).

03 Finding a duplicate

The slot that gets two visitors ▸

Flip the problem: now every value is in 1..n and exactly one value appears twice, while all others appear once. The missing-number and all-disappeared variants told us the missing slot — this variant tells us the doubled value. After the cyclic sort pass, the duplicate is the value sitting at the wrong index: the first position i where a[i] ≠ i + 1 gives us a[i] as the duplicate (because some other value that should be there was displaced by the repeat).

The brute-force alternatives are: sort and scan for adjacent equals at O(n log n), or use a hash set at O(n) space. The cyclic-sort gives O(n) time, O(1) space — and it does mutate the array, which is worth flagging to the interviewer before coding.

Cyclic sort (O(1) space)

  • O(n) time, O(1) space. Works if mutation is allowed.
  • Straightforward two-phase structure.
  • Mutates the array — flag this before coding.

Hash set (O(n) space)

  • O(n) time, O(n) space.
  • Works on unsorted, non-mutated input.
  • Not suitable when extra space is forbidden.
Worked problemMediumLeetCode 287 (variant A)
Find the Duplicate Number — cyclic sort approach

An array of n + 1 integers has every element in 1..n, and exactly one value appears more than once. Return that repeated value. You may not modify the array and must use O(1) extra space — but first, let us understand the sorting approach.

Think it through

Restate

n + 1 slots, values 1..n, one repeat. Find the duplicate. (The stricter form of the problem forbids mutation; we solve it first with sorting, then show Floyd's cycle in section 05.)

Brute force

Nested loop — for each i, check all j > i for equality. O(n²) time. Sorting then scanning is O(n log n). Hash set is O(n) time, O(n) space.

The waste

The hash set re-stores information the array encodes. After a cyclic sort, the duplicate is the value sitting at the wrong index — we just need to identify that misfit.

The move — phase 1

Cyclic sort with range 1..n: while a[i] ≠ i + 1, let home = a[i] − 1. If a[home] = a[i] the home is already occupied by the same value — this is the duplicate; return it. Otherwise swap.

The move — phase 2

(If phase 1 never caught it inline, scan for the first index where a[i] ≠ i + 1 and return a[i]. In practice the inline catch is enough for this variant.)

Invariant & cost

A duplicate blocks its home slot; we detect the collision immediately. O(n) time, O(1) space — but note: the array is mutated.

The code

python

def findDuplicateCyclicSort(nums):
    i = 0
    while i < len(nums):
        home = nums[i] - 1
        if nums[i] != i + 1:        # not yet home
            if nums[home] == nums[i]:  # home slot already has our value
                return nums[i]         # that's the duplicate
            nums[i], nums[home] = nums[home], nums[i]
        else:
            i += 1
    return -1                        # unreachable if input is valid
Narrate it to the interviewer

"Values are 1..n in an n+1-element array, so each value has a home index at value − 1. I cyclic-sort: at each position I try to send the current value home. If the home slot already holds the same value, I've found the duplicate — return it immediately. Otherwise I swap and retry. The collision can only happen with a repeated value. O(n) time, O(1) space — the trade-off is that I mutate the array. For the no-mutation O(1) space requirement, I'll use Floyd's cycle detection on the index-as-function graph, which I'll walk through next."

Dry-run aloud: [1, 3, 4, 2, 2] (duplicate 2); [3, 1, 3, 4, 2] (duplicate 3); [2, 2, 2, 2, 2] (all same, caught on first attempt to swap home).

04 No mutation

Treating the array as a function

The cyclic-sort approach to Find the Duplicate Number mutates the input — swapping values in place. The official problem statement (LeetCode 287) adds the constraint that you must not modify the array. Is O(1) space still achievable?

Yes — through a completely different lens. Treat each index i as a node and each value a[i] as the edge to the next node. Because values are in 1..n and there are n + 1 nodes (indices 0..n), by pigeonhole at least two indices share the same a[i] value, meaning two different nodes point to the same successor — a structural cycle. The entry point of that cycle is precisely the duplicate value. This is the same Floyd's tortoise-and-hare cycle detection you used on linked lists (see Lesson 11), now applied to an implicit graph. We walk through this in the interactive diagram of section 05.

Index-as-function vocabulary

Define f(x) = a[x]. Starting from index 0, repeatedly apply f: 0 → a[0] → a[a[0]] → …. Because the range 1..n always maps back into valid indices, this sequence never terminates — it must cycle. The duplicate value is the index that is pointed to twice, which is the cycle's entry node. Floyd's algorithm finds it in O(n) time, O(1) space, without touching the array.

05 Floyd on indices

Finding the cycle entry — no mutation ▸

The implicit graph from section 04 has a cycle whose entry is the duplicate value. Floyd's tortoise-and-hare algorithm (which you built intuition for on linked lists in Lesson 11) applies directly: let slow apply f once per tick and fast apply it twice. They must meet somewhere inside the cycle. Then reset slow to 0 and advance both one step per tick — they will converge at the cycle entry, which is the duplicate.

Both phases are O(n) time (the tail has at most n steps before the meeting, and phase 2 requires at most n steps to reach the entry). Space is O(1) — only two integer pointers. The array is never written to.

Worked problemHardLeetCode 287 (variant B)
Find the Duplicate Number — Floyd's cycle

An (n + 1)-element array contains integers each in 1..n, with exactly one duplicate. Return it using O(1) extra space without modifying the array.

Think it through

Restate

Same array as before — but now mutation is forbidden. I need O(1) space, O(n) time, read-only access.

Brute force

Nested loop O(n²), or sort O(n log n) and scan (requires mutation or O(n) copy), or hash set O(n) space. None meet all constraints.

The waste

All those approaches ignore the implicit graph structure: treating f(x) = a[x] as a function creates a sequence that must cycle (there are n+1 nodes but only n possible successors). Floyd's algorithm traverses this graph using only two integer variables.

The move — phase 1 (meet)

Start both slow and fast at index 0. Each tick: slow = a[slow]; fast = a[a[fast]]. Continue until they are equal — they will meet inside the cycle.

The move — phase 2 (locate)

Reset slow = 0. Now advance both one step at a time: slow = a[slow]; fast = a[fast]. They meet at the cycle entry — the duplicate value.

Invariant & cost

The cycle must exist (pigeonhole); the meeting inside the cycle is guaranteed; the phase-2 convergence on the entry follows from the same distance argument as linked-list Cycle II (Lesson 11). O(n) time, O(1) space, array untouched.

The code

python

def findDuplicate(nums):
    # phase 1: find a meeting point inside the cycle
    slow = nums[0]
    fast = nums[nums[0]]
    while slow != fast:
        slow = nums[slow]
        fast = nums[nums[fast]]
    # phase 2: find the cycle entry (the duplicate)
    slow = 0
    while slow != fast:
        slow = nums[slow]
        fast = nums[fast]
    return slow
Narrate it to the interviewer

"The no-mutation, O(1)-space constraint rules out sorting and hash sets. The key insight is that treating a[x] as a function from index to index creates an implicit linked-list-like structure that must cycle — because n+1 nodes map into only n possible successor values. The cycle entry is the duplicate. I apply Floyd's two-phase cycle detection: phase one has slow and fast meet inside the cycle; phase two resets slow to 0 and advances both by one step until they meet at the entry — the duplicate. O(n) time, O(1) space, no writes."

Dry-run aloud: [1, 3, 4, 2, 2] → f chain: 0→1→3→2→4→2→4→… (cycle at 2). [3, 1, 3, 4, 2] → f chain: 0→3→4→2→3→4→… (cycle at 3). Confirm phase 2 logic using the distance argument if the interviewer probes.

What the interviewer asks next

"Why do slow and fast both start at a[0] not at 0?" — Because index 0 is not reachable from any node (values are 1..n, so no a[i] equals 0), meaning 0 is purely the tail start. Starting at a[0] puts both pointers inside the reachable range immediately. "What if there are multiple duplicates?" — Floyd's approach only works for exactly one duplicate (the problem guarantees this); for multiple missing/extra values, use the cyclic-sort scan approach from sections 02-03.

06 Practice

Practice ladder

The cyclic-sort pattern is narrow but extremely reliable in interviews — once you spot the constrained value range, the two-pass structure is nearly mechanical. Work through these drills by stating the brute force first, then the cyclic-sort move, then coding it. Aim for under 20 minutes on Easy, under 35 on Medium.

EasyMissing Numbercyclic sort + scan

n distinct values from 0..n in an n-element array — find the one absent value.

Idea Cyclic sort with home = value (range 0..n); scan for first index where a[i] ≠ i; return n if all settled. O(n) / O(1).

EasyFind All Numbers Disappeared in an Arraycyclic sort + scan

Values 1..n in an n-element array, with duplicates; return every value that does not appear.

Idea Cyclic sort (home = value − 1); collect all indices where a[i] ≠ i + 1 into the output list. O(n) / O(1) aux.

EasySet Mismatchcyclic sort + two reads

An n-element array with values 1..n has exactly one duplicate and one missing value; return both.

Idea Cyclic sort; scan: where a[i] ≠ i + 1, the duplicate is a[i] and the missing is i + 1. One scan returns both. O(n) / O(1).

EasyFirst Missing Positivecyclic sort · range adaptation

Given an unsorted array of arbitrary integers (may include negatives), find the smallest positive integer not present.

Idea Ignore values outside 1..n; cyclic sort the rest (home = value − 1, skip if out of range); scan for first gap. O(n) / O(1).

Idea Cyclic sort; scan: wherever a[i] ≠ i + 1, a[i] is a duplicate. Collect into result list. O(n) / O(1).

MediumLinked List Cycle IIfloyd cycle entry

Find the node where a linked list's cycle begins, using O(1) space.

Idea Classic Floyd on nodes (see Lesson 11). The same two-phase reasoning: meet inside the cycle, then reset and converge on the entry. O(n) / O(1).

MediumHappy Numberimplicit cycle detection

A number is happy if repeatedly summing its digit-squares eventually reaches 1; return whether a given number is happy.

Idea Unhappy numbers cycle forever; treat digit-square-sum as f(x). Apply Floyd: if slow and fast meet at 1 the number is happy; any other meeting is the cycle. O(log n) digits / O(1).

HardFind the Missing Rangesrange scan

Given a sorted integer array and bounds [lo, hi], return every range of consecutive integers that is fully absent.

Idea Single linear scan with a pointer tracking the "expected next" value; emit a range whenever there is a gap. O(n) / O(1).

HardFirst Missing Positive (follow-up)negative marking variant

Same as above, but instead of cyclic sort, solve it by using negative signs as presence markers in the array itself.

Idea Two-pass: mark a[v−1] negative for every valid v seen; then scan for the first positive — its index + 1 is the answer. O(n) / O(1). Compare trade-offs with cyclic sort.

How to drill this pattern

For each problem:
(1) confirm the value range aloud — "values are in 1..n, so the array IS the hash table";
(2) state whether the variant needs a single missing, all missing, or the duplicate — this picks which scan you run after sorting;
(3) code the cyclic sort loop first, then the scan;
(4) dry-run three edge cases: all values present (no gaps), all values identical (n−1 gaps), and an array of size 1. For Floyd problems, also trace the distance argument once from memory to cement why phase 2 lands at the cycle entry.

★ Review

Summary & self-test

The whole lesson compresses to one observation: when values are constrained to a contiguous integer range, let the array serve as its own hash table — each value has a home index, and a single sorting pass followed by a single scan answers any missing-or-duplicate question in O(n) time and O(1) space. The harder variant (no mutation, one duplicate) re-uses Floyd's cycle detection from the linked-list world, applied to an implicit function graph.
