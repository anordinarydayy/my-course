Topological Sort

When tasks, courses, or build steps depend on each other, you need an ordering where every dependency comes before the thing that needs it. Topological sort turns a directed acyclic graph into a linear sequence where every edge points forward — and, as a bonus, tells you immediately when no such ordering can exist because a cycle makes it impossible.

00 The idea

What problem does it solve?

Suppose you are registering for university courses. Math 201 requires Calculus I; Calculus I requires Algebra. You cannot enroll in Math 201 until both prerequisites are done. More generally, a pile of tasks with directed "must come before" constraints defines a directed graph where an edge A → B means "A must be completed before B." The question is: can these tasks be ordered so that every edge points from an earlier task to a later one?

That linear ordering is called a topological order of the graph. It exists if and only if the graph has no directed cycle — a cycle would mean task A depends on B depends on A, making it impossible to start anywhere. Such a graph is called a DAG: a directed acyclic graph.

  • Course scheduling — register courses in prerequisite order (Course Schedule II, LeetCode 210).
  • Build systems — compile source files so every dependency is built before its dependents (Makefile order).
  • Alien dictionary — infer letter ordering from a sorted alien word list, then topologically sort the letters (LeetCode 269).
  • Package installers — install libraries before the packages that depend on them.
Two algorithms, same guarantee

There are two standard ways to compute topological order: Kahn's algorithm (BFS-based, iteratively peels zero-in-degree nodes) and DFS post-order (recursively explores and prepends finished nodes). Both run in O(V + E) time. Kahn's naturally detects cycles — if the output is shorter than the graph, nodes stuck in a cycle were never emitted. DFS detects cycles via a "currently visiting" colour. Interviewers most often expect Kahn's; knowing DFS post-order shows depth.

01 Kahn's algorithm

Peel the graph wave by wave ▸

The brute force is to try every permutation of nodes and check whether each is a valid topological order — that is O(V! · E), completely impractical for any non-trivial graph. The redundant work is obvious: we are blindly guessing orderings when the graph already tells us exactly which nodes are ready to go.

Kahn's insight is to process what is provably safe to emit right now: any node with in-degree zero has no remaining prerequisites, so it can come next in the ordering. Here is the procedure:

  • Step 1 — compute in-degrees. For each node, count how many edges point into it.
  • Step 2 — seed the queue. Push every node whose in-degree is currently zero.
  • Step 3 — peel. Pop a node from the queue, append it to the result, then decrement the in-degree of every neighbour it points to. If a neighbour's in-degree drops to zero, it is now unblocked — add it to the queue.
  • Step 4 — repeat until the queue is empty.

The invariant: a node enters the queue exactly when all of its predecessors have been emitted, so by the time you pop and emit it, no unfinished dependency exists. Each node and each edge is processed once: O(V + E) time, O(V) space for the queue and in-degree table.

Kahn's (BFS-based)

  • Naturally detects cycles: if result length < V, a cycle exists.
  • Easy to reason about — you always emit something provably ready.
  • Can process nodes in lexicographic order by using a min-heap instead of a queue.

DFS post-order

  • Finishes a node only after all its descendants are done, then prepends it.
  • Familiar if you are comfortable with recursive DFS.
  • Detects cycles with a three-colour state: unvisited, in-stack, done.
Worked problemMediumLeetCode 210
Course Schedule II

You have n courses numbered 0 to n−1 and a list of prerequisite pairs [a, b] meaning course b must be finished before course a. Return a valid order in which to take all the courses, or an empty list if no valid ordering exists.

Think it through

Restate

Build the DAG from the prerequisite pairs, then return a topological order of all V nodes, or [] if there is a cycle.

Brute force

Try all n! orderings and check each one against every prerequisite. Even for n = 10 that is 3.6 million checks. O(V! · E) — completely impractical.

The waste

We are guessing orderings even though the graph hands us the answer wave by wave. A node with in-degree zero is always safe to emit next — we never need to guess.

The move — Kahn's BFS

Build an adjacency list and an in-degree array from the prerequisite pairs (remember: pair [a,b] means edge b → a). Seed a queue with every node whose in-degree is 0. While the queue is non-empty: pop a node, append to result, decrement each neighbour's in-degree, push neighbours whose in-degree just hit 0. If the result holds all V nodes, return it; otherwise a cycle was detected, return [].

Invariant & cost

A node enters the queue only after all predecessors are in the result. Each node is popped once; each edge is traversed once: O(V + E) time, O(V + E) space.

The code

python

from collections import deque

def findOrder(numCourses, prerequisites):
    adj = [[] for _ in range(numCourses)]
    indeg = [0] * numCourses
    for a, b in prerequisites:      # b must come before a  →  edge b→a
        adj[b].append(a)
        indeg[a] += 1

    q = deque(i for i in range(numCourses) if indeg[i] == 0)
    order = []
    while q:
        node = q.popleft()
        order.append(node)
        for nei in adj[node]:
            indeg[nei] -= 1
            if indeg[nei] == 0:
                q.append(nei)

    return order if len(order) == numCourses else []  # [] → cycle
Narrate it to the interviewer

"Restating: build a DAG from the prerequisite pairs, then find a topological order of all n courses — or return an empty list if a cycle exists. A brute force tries all orderings, which is n! — completely impractical. Instead I'll use Kahn's: build adjacency list and in-degree array, seed a queue with the zero-indegree nodes, then peel: pop a node, add it to the result, decrement its neighbours' in-degrees, enqueue anything that drops to zero. At the end, if the result has all n courses I return it; if it's shorter, some nodes were stuck in a cycle and I return empty. Every node and every edge is visited once: O(V + E) time and space."

Dry-run aloud: n=1, no prerequisites (trivially returns [0]); n=2, prerequisites [[0,1],[1,0]] — a mutual cycle, result stays length 0, return []. Also check that pair [a,b] means b→a (easy to flip the edge direction).

02 DFS alternative

DFS post-order reversed

The second way to compute topological order leans on a recursive DFS. The key observation is: when DFS has fully explored every descendant of a node and is about to return, that node has no later dependency left to discover — it is safe to prepend it to the result. Collecting nodes in the order they finish and then reversing the list gives a valid topological order.

Concretely, keep a visited set (never revisit) and a result list. For each unvisited node, run DFS: recurse into all neighbours first, then after all descendants finish, append the current node. When DFS completes over all nodes, reverse the result.

Why the reversal? Node A finishes only after all nodes reachable from A finish, so A appears later in the finish list. After reversing, A is earlier than all its descendants — exactly the topological requirement. The algorithm still runs in O(V + E) because each node and edge is visited once.

The reusable shape

DFS post-order is useful when the graph arrives as an adjacency list and you are already inside a DFS traversal (for example, finding strongly connected components or detecting back edges). For plain topological sort at an interview, Kahn's is usually cleaner to write and easier to explain because cycle detection falls out for free without any extra colour state.

Worked problemMediumLeetCode 207
Course Schedule

You have n courses and a list of prerequisite pairs. Decide whether it is possible to finish all courses — equivalently, whether the prerequisite graph contains a directed cycle.

Think it through

Restate

Return True if the directed graph of prerequisites is acyclic (a valid topological order exists), False otherwise.

Brute force

DFS from every node, tracking visited nodes in a set to catch re-visits. But a plain visited set does not distinguish "visited in an earlier DFS call" from "currently on the DFS stack" — we need the latter to spot a back edge (a cycle). A set per call works but costs O(V · (V + E)).

The waste

Re-doing work for already-confirmed-acyclic subtrees. We can use three states — unvisited, in-stack, done — so each node is fully analysed once and its subtree's result is cached.

The move — DFS three-colour

0 = unvisited, 1 = currently on the DFS stack (grey), 2 = fully explored (black). For each unvisited node, DFS: mark grey, recurse into neighbours; if a neighbour is grey, we found a back edge → cycle → return False; mark black on return. If no cycle is found over all nodes, return True.

Invariant & cost

A grey-to-grey edge is a back edge, which exists if and only if the graph has a cycle. Each node is processed exactly once: O(V + E) time, O(V) space for the colour array and recursion stack.

The code

python

def canFinish(numCourses, prerequisites):
    adj = [[] for _ in range(numCourses)]
    for a, b in prerequisites:
        adj[b].append(a)

    color = [0] * numCourses  # 0 unvisited, 1 in-stack, 2 done

    def dfs(u):
        if color[u] == 1: return False  # back edge → cycle
        if color[u] == 2: return True   # already confirmed acyclic
        color[u] = 1
        for v in adj[u]:
            if not dfs(v): return False
        color[u] = 2
        return True

    return all(dfs(i) for i in range(numCourses) if color[i] == 0)
Narrate it to the interviewer

"This is just 'is the prerequisite graph a DAG?' — if it has a cycle, some courses are mutually dependent and we can never start. I'll use three-colour DFS: 0 = unvisited, 1 = currently on the stack, 2 = fully explored. When I visit a node I mark it grey and recurse into its neighbours; if I reach a grey node I've found a back edge — that's a cycle, return False. After all neighbours are clean I mark it black. Starting DFS from every unvisited node ensures the whole graph is checked. No cycle found → return True. Each node and edge is processed once: O(V + E) time, O(V) space."

Dry-run aloud: a single self-loop (adj[0]=[0], grey→grey immediately → cycle); n=2 with no prerequisites (both DFS calls return True immediately); the pair [0,1],[1,0] forming a mutual cycle.

03 Cycle detection

When the output is too short, a cycle lurks ▸

Kahn's algorithm exposes cycles effortlessly. Once the queue empties, every node that has been emitted had its in-degree reach zero during the run. Any node that still has a positive in-degree was never unblocked — it sits inside a cycle where each member is still waiting on another member.

The test is a single line: if len(result) < V, there is a cycle. Nodes trapped in a cycle never enter the queue, so they are never emitted. The size of the shortage tells you how many nodes are in cycles (or transitively depend on cycles). To reconstruct which nodes are cyclic, collect the set of all nodes minus the emitted ones.

DFS achieves the same check differently: during the DFS, if you reach a node that is currently on the call stack (state = in-stack), you have found a back edge, which is the defining feature of a cycle in a directed graph.

Cycle detection is built in

Never add a separate DFS cycle-check before running Kahn's. The single comparison len(result) == V at the end does both jobs — it returns the order when one exists and flags the cycle when it doesn't. Adding a pre-check doubles the work without gaining anything.

Worked problemHardLeetCode 269
Alien Dictionary

You receive a list of words written in an alien language, sorted lexicographically by that language's alphabet. Derive the alphabetical ordering of the alien letters. If no valid ordering exists (the input contains a contradiction), return an empty string. If multiple valid orderings exist, return any one of them.

Think it through

Restate

Extract letter-ordering edges from adjacent word pairs, build a DAG of letters, then topologically sort it. If a cycle appears or the input is contradictory, return "".

Brute force

Try all permutations of the letters and check each against every derived ordering constraint. With up to 26 letters that is 26! permutations — completely infeasible.

The waste

We do not need to guess orderings. Each pair of consecutive words in the sorted list gives us at most one edge: compare character by character, and the first position where they differ tells us which letter comes earlier in the alien alphabet.

The move — three phases

Phase 1 — collect letters: every character that appears in any word is a node in the graph. Phase 2 — extract edges: for each consecutive pair of words, zip their characters until the first mismatch; that pair (c1, c2) gives edge c1 → c2. Special case: if word1 is a prefix of word2 but word1 appears after word2 in the list, that is an immediate contradiction — return "". Phase 3 — Kahn's: topological sort the letter graph; if result length equals the total number of distinct letters, join and return; else a cycle → return "".

Invariant & cost

At most 26 letters, so V ≤ 26. Edge extraction visits each character once: O(total characters in all words). Kahn's over the letter graph: O(V + E). Overall O(C) where C is the total character count, since V and E are at most 26 and 26² respectively.

The code

python

from collections import deque, defaultdict

def alienOrder(words):
    # collect all distinct letters as nodes
    adj = defaultdict(set)
    indeg = {c: 0 for w in words for c in w}

    # extract ordering edges from consecutive word pairs
    for i in range(len(words) - 1):
        w1, w2 = words[i], words[i + 1]
        min_len = min(len(w1), len(w2))
        if w1[:min_len] == w2[:min_len] and len(w1) > len(w2):
            return ""             # prefix contradiction
        for c1, c2 in zip(w1, w2):
            if c1 != c2:
                if c2 not in adj[c1]:
                    adj[c1].add(c2)
                    indeg[c2] += 1
                break         # only the first mismatch gives info

    # Kahn's topological sort over the letter graph
    q = deque(c for c in indeg if indeg[c] == 0)
    result = []
    while q:
        c = q.popleft()
        result.append(c)
        for nei in adj[c]:
            indeg[nei] -= 1
            if indeg[nei] == 0:
                q.append(nei)

    return "" if len(result) != len(indeg) else "".join(result)
Narrate it to the interviewer

"There are three phases. First I collect every distinct letter across all words as a node. Second I extract ordering edges: for each consecutive pair of words I zip them character by character until the first mismatch — that first differing pair tells me which letter comes earlier. I also check for the prefix contradiction: if word1 is a prefix of word2 but word1 appears after word2, that is impossible, return empty immediately. Third I run Kahn's on the letter graph. If the result covers all letters I join and return; if it's shorter, a cycle means the ordering is contradictory. Everything is O(total characters) since the letter graph has at most 26 nodes."

Dry-run aloud: a single word (no edges, any letter order is valid, return the letters in any order); two identical words (no mismatch edge, still valid); ["ab","a"] — word1 is a prefix of word2 but word1 is longer reversed — this is the prefix contradiction, return "".

What the interviewer asks next

"What if there are multiple valid orderings?" → use a min-heap instead of a plain queue so you always emit the lexicographically smallest available letter, giving the unique smallest valid order. "Can you tell which nodes are in cycles?" → collect the set of all nodes minus the emitted result. "What if the graph is very large but sparse?" → still O(V + E), which is optimal since you must visit every node and edge at least once. "Reconstruct the actual cycle" → switch to DFS and record the path when a back edge is found.

04 Applications

Course schedule, build order, and beyond

Topological sort shows up whenever a system has explicit "before-after" rules. The DAG structure is the invariant — as long as dependencies are acyclic, the algorithm works identically regardless of what the nodes represent. Here is how the same skeleton maps to three important families:

  • Course scheduling (LeetCode 207, 210). Nodes are courses, edges are prerequisite pairs. 207 asks "is it possible?" — just check for cycles. 210 asks "in what order?" — return Kahn's output. The only tricky part is reading the pair [a,b] correctly: pair [a,b] means b is a prerequisite of a, so the edge goes b → a.
  • Build systems. Nodes are source files or compilation units; a directed edge from X to Y means Y must be compiled before X. A Makefile is essentially a serialised topological order. If a cycle is found the build system cannot proceed — it reports a "circular dependency" error, which is exactly Kahn's result of length < V.
  • Alien dictionary (LeetCode 269). Nodes are characters in the alien alphabet; an edge c1 → c2 means c1 comes before c2. The challenge is constructing the graph from word comparisons before running the standard topological sort. The special cases (prefix contradiction, multiple valid orderings) are all about graph construction, not the sort itself.
The decision: when to reach for topological sort

Reach for topological sort when a problem has explicit ordering constraints (task A must come before B) or when you are asked to sequence items from a directed dependency structure. If the phrase "prerequisite", "dependency", "before/after", or "valid order" appears — think topological sort. If the graph might have cycles and you need to detect them — topological sort still applies via the output-length check.

05 Graph construction

Alien dictionary — building the graph ▸

The hard part of the alien dictionary problem is not the topological sort — it is the edge extraction step. Given a list of words sorted in alien lexicographic order, you compare adjacent words and extract constraints on the letter ordering. Walk through this carefully because the edge cases here are where candidates lose points.

For each consecutive pair (word1, word2), zip the characters in parallel. The first position where they differ gives you one directed edge: word1's character comes before word2's character in the alien alphabet. That is the only information you can extract from this pair — even if later characters differ, they add no new constraint because the first difference already determined the relative order of these two words.

Two special cases to handle before running the sort:

  • Prefix contradiction. If word1 starts with word2 (word2 is a prefix of word1) but word1 appears before word2 in the sorted list, the input is impossible — return "". Example: ["apple", "app"] — "apple" can never come before "app" in a correct sorted order.
  • No first mismatch. If word1 and word2 are identical, they produce no edge — skip them.
What the interviewer asks next

"How many edges can the graph have?" → at most O(V · E) from word comparisons, but since V ≤ 26 letters and each word pair gives at most one edge, the edge count is bounded by the number of word pairs. "What if all words are the same?" → no edges, every letter ordering is valid, return any ordering of the distinct letters. "How does the prefix contradiction differ from a cycle?" → a prefix contradiction is caught during edge extraction before Kahn's runs; a cycle is detected by Kahn's output being shorter than the node count.

06 Practice

Practice ladder

The pattern solidifies only through repetition. For each problem below, narrate the brute force and its complexity first, identify why topological sort applies, write the code, and dry-run at least two edge cases. Aim for the Easy tier in under 15 minutes and the Medium tier in under 30.

EasyCourse Schedulecycle detection

Given n courses and prerequisite pairs, decide if all courses can be completed — i.e., is the dependency graph a DAG?

Idea Run Kahn's; if result length = n, return True. O(V + E).

EasyFind Eventual Safe Statesreverse graph

A node in a directed graph is "safe" if every path from it eventually reaches a terminal node. Return all safe nodes in sorted order.

Idea Reverse all edges; terminal nodes become in-degree-zero sources; run Kahn's on the reversed graph — emitted nodes are the safe states.

EasyMinimum Height Treesiterative leaf pruning

Find all roots of an undirected tree that minimise the tree's height — there are at most two such roots.

Idea Iteratively remove leaf nodes (degree 1) until 1 or 2 nodes remain — identical logic to peeling zero-in-degree nodes in Kahn's.

MediumCourse Schedule IIKahn's full output

Return any valid ordering of all n courses, or an empty list if a cycle exists.

Idea Kahn's algorithm; output is the result list. O(V + E).

MediumSequence Reconstructionunique topo order

Determine whether a target sequence is the one and only shortest supersequence that can be reconstructed from a set of shorter sequences.

Idea Build a DAG from the shorter sequences; at every Kahn's step, the queue must have exactly one element — any other size means the topological order is not unique.

MediumLongest Path in a DAGtopo + DP

Given a DAG with weighted edges, find the length of the longest path from any source to any sink.

Idea Topological order first; then process nodes left-to-right, relaxing outgoing edges — a single DP pass replaces Dijkstra because there are no back edges.

MediumParallel Courses IImin semesters + bitmask

Find the minimum number of semesters to finish all courses when you may take at most k courses per semester and prerequisites must be met.

Idea Bitmask DP over subsets; the prerequisite check is a topological constraint — nodes can only be selected when all predecessors are in the chosen set.

MediumBuild Order (Classic)Kahn's + detection

Given a list of projects and dependency pairs (a, b) meaning b must be built before a, return any valid build order, or signal that no order exists.

Idea Exact Course Schedule II — build adjacency list, run Kahn's, check for cycle. Watch the edge direction: (a,b) means edge b→a.

HardAlien Dictionarygraph construction + topo

Given a sorted alien word list, return the alphabetical ordering of the alien letters, or "" if a contradiction exists.

Idea Extract one edge per adjacent word pair (first mismatch); check prefix contradiction; run Kahn's; detect cycle by output length.

HardSort Items by Groups Respecting Dependenciestwo-level topo

Items belong to groups; items and groups both have ordering constraints. Return a valid item ordering respecting all constraints.

Idea Run topological sort twice: once within each group to order items, once over groups to order groups relative to each other. A cycle in either level → return [].

HardParallel Courseslevel-by-level BFS

All available courses are taken simultaneously each semester; find the minimum number of semesters to complete all courses.

Idea Kahn's level by level: each BFS wave is one semester. Count the waves. If any node is never emitted (cycle), return -1.

How to drill topological sort problems

Out loud:
(1) identify the nodes and edges from the problem statement;
(2) state whether Kahn's or DFS post-order is cleaner — usually Kahn's;
(3) trace through the in-degree computation and the queue seed;
(4) state the cycle check;
(5) code it;
(6) dry-run edge cases: a single node with no edges, a graph with a self-loop, two mutually dependent nodes, and an already-linear chain. For alien-dictionary variants, also check the prefix-contradiction case before touching the sort.

★ Review

Summary & self-test

The whole pattern compresses to one sentence: if tasks have directed dependencies and no cycles, Kahn's BFS or DFS post-order both produce an ordering in O(V + E) where every prerequisite comes first — and both detect cycles for free. Master the two algorithms, the cycle-check idiom, and the edge-extraction step for graph-construction problems, and this family becomes a single recognisable template.

(2) Why does a node with in-degree zero always exist in a DAG, but not necessarily in a graph with cycles?
