Shortest Path — Dijkstra & 0-1 BFS

When edges carry weights, plain BFS no longer finds the shortest path — you need an algorithm that always expands the currently cheapest known node first. Dijkstra does exactly that with a min-heap; 0-1 BFS replaces the heap with a deque when weights are only 0 or 1; and Bellman-Ford handles the hardest case of negative edges. All three encode the same greedy-relaxation idea, and each one appears repeatedly in graph interview problems.

00 The problem family

Weighted shortest paths

Plain BFS finds the path that crosses the fewest edges — it treats every edge as costing 1. As soon as edges have different weights, the fewest-hops path and the least-cost path are different things. Consider roads between cities: the most direct route (fewest turns) is not necessarily the fastest (fewest minutes). You need an algorithm that keeps track of the total accumulated cost to each node and always expands the cheapest one next.

The family of problems you will see in interviews:

  • Single-source shortest paths — from one node, what is the minimum cost to reach every other? Network Delay Time (743).
  • Shortest path under a constraint — fewest cost, but with an extra limit like "at most K intermediate hops". Cheapest Flights Within K Stops (787).
  • Min-cost path in a grid — each cell has a cost and you need to cross the grid cheaply. Path With Minimum Effort (1631).
  • 0/1-weight graphs — edges cost either 0 or 1; a deque is faster than a heap. Minimum Obstacle Removal (2290).

All three algorithms (Dijkstra, 0-1 BFS, Bellman-Ford) share the same core operation: relaxation. If the known distance to node v is dist[v], and you find an edge from u to v with weight w, you check whether dist[u] + w < dist[v]. If so, you found a cheaper route and you update it.

The relaxation step

Every shortest-path algorithm is built around one primitive: if I just discovered a cheaper way to reach node v, update its known distance and schedule it for re-expansion. The algorithms only differ in the order they pop nodes from their frontier data structure — heap, deque, or simple queue over rounds.

01 Dijkstra

Min-heap: always expand the cheapest known node ▸

The brute-force answer is to try every path from source to every other node and take the minimum — an exponential search on a general graph. Even with dynamic programming that is O(V·E) in the worst case without a smarter frontier.

Dijkstra's insight is a greedy one: at any point during the search, the node with the smallest known distance cannot have a shorter path discovered later — as long as all edge weights are non-negative. So you can safely settle (mark as final) the cheapest node in your frontier, then relax all its outgoing edges. Doing this with a min-heap gives you the cheapest unsettled node in O(log V) time.

The algorithm in plain English:

  • Initialize dist[source] = 0, all others = infinity. Push (0, source) into a min-heap.
  • Pop the minimum (cost, u). If u is already settled, skip it (stale heap entry).
  • Mark u settled. For each neighbor v with edge weight w: if dist[u] + w < dist[v], update dist[v] and push (dist[v], v) onto the heap.
  • Repeat until the heap is empty or the target is settled.

Time: O((V + E) log V) with a binary heap. Space: O(V + E) for the graph and dist array.

Dijkstra (non-negative weights)

  • O((V+E) log V) with a binary heap.
  • Greedy: each pop is permanently correct — no revisiting.
  • Fails on negative edges; the greedy assumption breaks.

Plain BFS (unit weights)

  • O(V+E) — no heap needed when all edges cost 1.
  • The frontier is a simple queue; each wave is one hop.
  • Use BFS when the graph is unweighted; reach for Dijkstra the moment weights appear.
Worked problemMediumLeetCode 743
Network Delay Time

You have a directed weighted network of n nodes and a list of travel-time edges. A signal is sent from a given source node. Return the time it takes for all nodes to receive the signal, or -1 if any node is unreachable.

Think it through

Restate

Find the shortest path from a single source to every other node in a directed weighted graph; the answer is the maximum of those shortest distances (the last node to get the signal). If any node stays at infinity, return -1.

Brute force

Run BFS from the source, but since weights differ we would need to re-expand nodes that receive an update — essentially Bellman-Ford: O(V × E) relaxations per round for V-1 rounds. That is O(V ⋅ E). With n up to 100 and edges up to 6,000 that is manageable, but Dijkstra is faster and more elegant.

The waste

Bellman-Ford redundantly revisits nodes even when their distance is already optimal. If all weights are non-negative (travel times are never negative), the greedy ordering lets us settle each node exactly once.

The move

Build an adjacency list. Run Dijkstra from the source: maintain a dist array initialized to infinity except dist[source] = 0; push (0, source) onto a min-heap. Each pop settles the nearest node and relaxes its neighbors. After the heap empties, the answer is max(dist) — if any value is still infinity, return -1.

Invariant & cost

When a node is popped from the heap, its distance is final (non-negative weights guarantee no shorter path can arrive later). Each node is settled once: O((V+E) log V) time, O(V+E) space for the graph and heap.

The code

python

import heapq
from collections import defaultdict

def networkDelayTime(times, n, k):
    graph = defaultdict(list)
    for u, v, w in times:
        graph[u].append((v, w))          # directed edge u->v with weight w

    dist = [float('inf')] * (n + 1)    # 1-indexed; dist[0] unused
    dist[k] = 0
    heap = [(0, k)]                      # (cost, node)

    while heap:
        d, u = heapq.heappop(heap)
        if d > dist[u]:                   # stale heap entry — skip
            continue
        for v, w in graph[u]:
            nd = d + w
            if nd < dist[v]:
                dist[v] = nd
                heapq.heappush(heap, (nd, v))

    ans = max(dist[1:])                   # dist[0] excluded (unused)
    return ans if ans < float('inf') else -1
Narrate it to the interviewer

"The problem is single-source shortest paths on a directed weighted graph. I want the maximum dist over all nodes, because the last one to receive the signal determines the total time. Brute force is Bellman-Ford at O(V·E). Since travel times are non-negative, I can use Dijkstra: initialize dist to infinity, set the source to 0, push it onto a min-heap. Each pop takes the cheapest unprocessed node and relaxes its neighbors — if we find a cheaper route to a neighbor, update and push. Stale heap entries are discarded when their stored cost is larger than the current dist. The invariant is that once a node is popped its distance is final. At the end, the max of all distances is the answer; if any is infinite, some node is unreachable. Time is O((V+E) log V), space O(V+E)."

Dry-run aloud: n=1 (the source is the only node, delay is 0); a graph where one node has no incoming edges from the source (return -1); all edges from the same source with the same weight.

What the interviewer asks next

"Can you reconstruct the actual shortest path?" → keep a prev array; when you update dist[v], set prev[v] = u, then walk back from the target. "What if nodes are 0-indexed?" → shift the 1-indexed dist array or use a dict. "Why skip stale entries instead of a visited set?" → heap can hold multiple entries for the same node; the first pop with the correct distance settles it; subsequent pops see d > dist[u] and bail — this is correct and avoids a separate set.

02 The limitation

Why negative edges break Dijkstra

Dijkstra's correctness rests on one assumption: once a node is popped from the heap, no cheaper path to it can ever be found. This is true when all edge weights are non-negative, because adding any edge can only increase or maintain a path's cost.

With a negative edge, a later expansion can reach a node through a path that costs less than what was recorded when that node was originally settled. The algorithm misses the update because it treated the node as finalized.

With weights S→A=2, S→B=5, A→T=4, B→T=−3, Dijkstra pops A first (dist 2), then relaxes T to 6. It then pops T (dist 6) before B (dist 5). When it finally processes B, T is already settled so the relaxation 5+(−3)=2 is ignored. The correct answer is 2, but Dijkstra reports 6.

Two alternatives handle negative weights. 0-1 BFS works when weights are restricted to exactly 0 or 1. Bellman-Ford handles arbitrary negative weights (but not negative-weight cycles, which make "shortest path" undefined).

Worked problemMediumLeetCode 787
Cheapest Flights Within K Stops

Given a set of directed flight routes with prices, find the cheapest way to fly from a source city to a destination, using at most K intermediate stops. Return -1 if no such route exists.

Think it through

Restate

Shortest path from source to destination in a directed weighted graph, but the path is limited to at most K+1 edges (K stops means K+1 flights). A standard Dijkstra without the stop constraint would find the cheapest overall; I need the cheapest that stays within the hop budget.

Brute force

DFS from source, exploring all routes of length at most K+1, taking the minimum cost. With up to 100 nodes and 10,000 edges this is exponential in the worst case — O(n^K) paths.

The waste

The DFS re-explores the same (node, stops-used) states repeatedly. State is both the current node and the number of stops consumed so far. Once I memoize those, the problem becomes a bounded-relaxation shortest path — exactly what Bellman-Ford's first K+1 rounds compute.

The move

Run a BFS-style relaxation for exactly K+1 rounds. In each round, try every edge (u, v, price): if prev[u] + price < curr[v], update. Use two arrays (prev and curr) so round-N updates do not bleed into round-N relaxations. After K+1 rounds, curr[dst] is the answer.

Invariant & cost

After round r, dist[v] holds the cheapest cost to reach v using exactly r edges. K+1 rounds covers all paths of at most K+1 edges. O(K ⋅ E) time, O(V) space.

The code

python

def findCheapestPrice(n, flights, src, dst, k):
    INF = float('inf')
    prev = [INF] * n
    prev[src] = 0

    for _ in range(k + 1):            # k stops = k+1 flights
        curr = prev[:]                  # copy: this round reads from prev
        for u, v, price in flights:
            if prev[u] < INF:          # u reachable in prev round
                curr[v] = min(curr[v], prev[u] + price)
        prev = curr

    return prev[dst] if prev[dst] < INF else -1
Narrate it to the interviewer

"This is cheapest path with an upper bound on edges — K stops means K+1 flights. Standard Dijkstra ignores the hop count, so I reach for bounded Bellman-Ford instead. I run K+1 rounds of relaxation; in each round I scan every edge and check if I can improve a destination's cost. The key detail is the two-array trick: I copy the previous distances at the start of each round and only read from the copy, so a round-N update can't be used to chain further within the same round — that would allow using more than one edge per round. After K+1 rounds the answer is dist[dst], or -1 if it's still infinity. Time is O(K·E), space O(n)."

Dry-run aloud: src == dst (return 0, k doesn't matter); no path exists (return -1); k=0 (direct flights only — the loop runs once and only direct src→dst edges count).

03 0-1 BFS

Deque: zero-cost edges go front, one-cost go back ▸

A special case of weighted shortest path arises often in grid problems: each step costs either 0 or 1. Using a full min-heap here would cost O((V+E) log V), but the structure can be exploited to achieve O(V+E) — linear time — with a double-ended queue (deque).

The idea mirrors Dijkstra's greedy: always expand the node with the smallest known distance first. Because weights are only 0 or 1, the frontier at any moment has at most two distinct distance levels — d and d+1. A deque can maintain this without sorting:

  • When you relax a neighbor via a 0-weight edge, the neighbor's cost doesn't increase — push it to the front of the deque so it is processed before anything at cost d+1.
  • When you relax a neighbor via a 1-weight edge, push it to the back — it belongs to the next distance level.

The deque naturally acts as a sorted frontier with only two priority levels, and you process nodes in non-decreasing distance order — just like Dijkstra, but without a heap.

The reusable shape

Any grid problem where cells have binary cost (free vs. blocked, blocked means cost 1 to remove) is 0-1 BFS territory. The template is three lines: build the dist grid (infinity), push the start onto the deque with dist 0, then pop from the left and for each neighbor choose appendleft (0-cost move) or append (1-cost move) based on whether you are crossing an obstacle.

Worked problemMediumLeetCode 1631
Path With Minimum Effort

You are crossing a grid of elevation heights from the top-left to the bottom-right. The effort of a path is the maximum absolute difference in height between any two consecutive cells on the path. Find the path with the minimum possible effort.

Think it through

Restate

Minimize the worst single step on any path from (0,0) to (rows-1, cols-1). The cost to reach a cell is the maximum height-difference edge along the best path to it so far.

Brute force

BFS/DFS all paths, tracking the maximum step cost along each. Exponential in the worst case — O(4^(rows·cols)) paths.

The waste

We re-explore cells reached by worse paths. This is a shortest-path problem where the "distance" of a cell is the max absolute difference on the best path to it — use Dijkstra with that custom distance metric.

The move

Build an effort grid initialized to infinity; set effort[0][0] = 0. Use a min-heap of (effort_so_far, row, col). When expanding a cell, the cost to enter a neighbor is max(current_effort, abs(height_diff)). Push the neighbor onto the heap if this is better than its recorded effort.

Invariant & cost

When a cell is popped, its effort is final (the min-heap guarantees we always process the best available path). O(rows ⋅ cols ⋅ log(rows ⋅ cols)) time, O(rows ⋅ cols) space.

The code

python

import heapq

def minimumEffortPath(heights):
    rows, cols = len(heights), len(heights[0])
    INF = float('inf')
    effort = [[INF] * cols for _ in range(rows)]
    effort[0][0] = 0
    heap = [(0, 0, 0)]                 # (effort, row, col)

    dirs = [(0,1),(0,-1),(1,0),(-1,0)]
    while heap:
        eff, r, c = heapq.heappop(heap)
        if eff > effort[r][c]:            # stale entry
            continue
        if r == rows-1 and c == cols-1:
            return eff                    # destination reached
        for dr, dc in dirs:
            nr, nc = r+dr, c+dc
            if 0 <= nr < rows and 0 <= nc < cols:
                step = abs(heights[nr][nc] - heights[r][c])
                ne = max(eff, step)       # bottleneck cost
                if ne < effort[nr][nc]:
                    effort[nr][nc] = ne
                    heapq.heappush(heap, (ne, nr, nc))
    return 0                             # 1×1 grid — start is destination
Narrate it to the interviewer

"I need to minimize the worst single step — that's not a sum but a max along the path. I can still model this as a shortest-path problem: the 'distance' to a cell is the minimum over all paths of the maximum step on that path. Dijkstra works here because larger costs never help — the metric is monotone. I initialize an effort grid to infinity, set the top-left to 0, and use a min-heap. When expanding, the cost to enter a neighbor is the max of the current path's worst step and the new height diff. I update if that's better than the recorded effort. The destination pops when its effort is final. O(R·C·log(R·C)) time, O(R·C) space. A 1×1 grid trivially returns 0."

Dry-run aloud: a 1×1 grid (return 0 immediately); a flat grid of equal heights (every step costs 0, answer is 0); a grid where every path passes through one extreme step (that step is unavoidable and will be the answer).

04 Bellman-Ford

Round-by-round relaxation for negative edges ▸

Bellman-Ford abandons the greedy order entirely. Instead of always expanding the cheapest node, it makes V-1 complete passes over all edges, relaxing every edge in every pass. Any shortest path can have at most V-1 edges (otherwise it would visit a node twice, implying a cycle), so V-1 rounds are sufficient to propagate the optimal distances from the source to every reachable node.

The key insight is that after round k, dist[v] holds the cost of the cheapest path to v using at most k edges. This is exactly why the Cheapest Flights problem uses K+1 rounds of Bellman-Ford — it naturally enforces the edge-count constraint.

An extra round (the V-th pass) tests for negative cycles: if any distance can still be improved, the graph contains a negative-weight cycle and shortest paths are undefined.

Bellman-Ford

  • O(V·E) time — slower than Dijkstra.
  • Handles negative edges; detects negative cycles.
  • The K+1-rounds variant enforces a hop-count budget (LeetCode 787).

Dijkstra

  • O((V+E) log V) — much faster in practice.
  • Only non-negative weights; no cycle detection.
  • Default choice when constraints are non-negative and K is not constrained.
Negative-cycle detection

After V-1 rounds, run one more pass. If any edge (u,v,w) still satisfies dist[u] + w < dist[v], there is a negative cycle reachable from the source. In practice, interview problems tell you the graph has no negative cycles; just confirm this aloud so the interviewer knows you're aware of the edge case.

05 Recognition

Spotting it, and the traps

The first decision is choosing the right algorithm. Here is the decision tree:

  • All edge weights non-negative, no hop limit → Dijkstra with a min-heap. O((V+E) log V).
  • Edge weights 0 or 1 only → 0-1 BFS with a deque. O(V+E).
  • All edges have the same weight (or unweighted) → plain BFS. O(V+E).
  • Hop-count constraint (at most K edges) → Bellman-Ford for K+1 rounds. O(K·E).
  • Negative edges, no hop limit → full Bellman-Ford (V-1 rounds). O(V·E).

Common signals in the problem statement:

  • "Minimum total cost", "shortest path", "minimum time to reach" with edge weights → Dijkstra.
  • "Grid", "cells can be blocked", "remove obstacle" → likely 0-1 BFS or Dijkstra on a grid.
  • "At most K stops", "using at most K edges" → bounded Bellman-Ford.
  • "Negative weights" or "penalties" that can reduce total cost → Bellman-Ford.

The bugs that sink candidates in this pattern:

  • Forgetting the stale-entry check in Dijkstra. Without if d > dist[u]: continue, you re-process settled nodes, which is wrong with negative edges (not applicable) and wasteful with non-negative ones. Always include it.
  • Using Dijkstra when there is a negative edge. The algorithm silently returns wrong answers — no crash, just wrong numbers. If in doubt, ask the interviewer.
  • Using the same dist array in both rounds of Bellman-Ford. In the K-stops variant you must copy prev before the round starts; otherwise a single round can chain multiple edges.
  • Off-by-one on K. K stops means K intermediate nodes, so K+1 edges, so K+1 rounds of Bellman-Ford. Double-check the problem's definition of "stop".
  • Forgetting to handle disconnected graphs. Always check if dist[dst] is still infinity at the end and return -1 in that case.
The algorithm selector in one glance

Look at the edge weights first. Non-negative and unconstrained → Dijkstra. Binary (0/1) → 0-1 BFS. Hop limit → bounded Bellman-Ford rounds. Negative edges → full Bellman-Ford. When the graph is a grid, Dijkstra with (row, col, cost) tuples in the heap is almost always the right shape.

Practice ladder

From a single source node in a directed weighted graph, find how long until all nodes receive a signal — or report that some node is unreachable.

MediumMinimum Cost to Make All Characters Equalgreedy / dp

Given a binary string, flip prefixes or suffixes; each flip of a substring of length k costs k. Find the minimum total cost to make all characters equal.

Idea Greedy: for each position, pay the min(left_len, right_len) when the character differs from the previous — no graph needed, but the cost-accumulation logic mirrors relaxation thinking. O(n).

Travel between cities; staying in a city overnight costs a nightly rate, and there are at most K nights of budget. Find the minimum cost to reach the destination.

Idea Expand state to (node, nights_used); Dijkstra over this state space. O((V·K) · E · log(V·K)).

HardNumber of Ways to Arrive at Destinationdijkstra · count paths

Find the number of shortest paths from node 0 to node n-1 in an undirected weighted graph, modulo 10^9+7.

Idea Dijkstra simultaneously tracking dist and ways[]; when dist[v] is improved, reset ways[v] = ways[u]; when dist[v] == dist[u]+w, add ways[u] to ways[v]. O((V+E) log V).

Given a stick of length n and a set of cut positions, the cost of a cut is the length of the current stick segment. Find the minimum total cost to make all cuts.

Idea Interval DP — not a graph problem, but reached by the same cost-minimization intuition. O(m³) where m is the number of cuts.

How to drill this pattern

For each problem:
(1) identify the graph structure (explicit edges, grid, implicit);
(2) state the edge weights and whether they are non-negative;
(3) choose the algorithm out loud with its complexity;
(4) write the template (heap/deque initialization, dist array, relax loop);
(5) code it;
(6) dry-run edge cases — single node, disconnected graph, negative edges (state whether they can appear), k=0 stops. The switch between Dijkstra, 0-1 BFS, and Bellman-Ford is the primary decision and should come before you touch the keyboard.

★ Review

Summary & self-test

Every shortest-path algorithm is a loop over the same primitive — relaxation — varied only in the order it picks which node to expand next. Dijkstra picks the globally cheapest unsettled node using a heap (greedy, correct only for non-negative weights). 0-1 BFS mimics that order with a deque when weights are binary, achieving linear time. Bellman-Ford gives up on clever ordering and simply repeats relaxation V-1 times, sacrificing speed to gain the ability to handle negative edges and bounded hop counts.

01RelaxationIf dist[u]+w < dist[v], update dist[v] and re-schedule v. The whole pattern is this one check.
02DijkstraMin-heap, always pop cheapest; correct for non-negative weights only. O((V+E) log V).
030-1 BFSDeque; 0-cost neighbors go front, 1-cost go back. Linear O(V+E) when weights are binary.
04Bellman-FordV-1 full edge passes; handles negative weights, detects negative cycles. O(V·E).
05K-stops = K+1 roundsBounded Bellman-Ford with a copy array per round enforces the hop-count constraint cleanly.
06Stale entriesIn Dijkstra, always skip if popped dist > recorded dist — heap may hold outdated entries.
07Grid DijkstraState is (cost, row, col); cost function can be sum, max, or any monotone metric over the path.
08Choose the algorithm firstWeights non-negative? Dijkstra. Binary? 0-1 BFS. Hop limit? Bounded B-F. Negative? Full B-F.
Self-test before you move on

Cover the notes and answer aloud:
(1) Why is Dijkstra's pop-order greedy correct for non-negative weights, and what precise assumption breaks with a negative edge?
(2) Walk through one step of 0-1 BFS: a cell with cost 3 is popped; it has a free neighbor and an obstacle neighbor — where does each go in the deque and why?
(3) After K+1 rounds of Bellman-Ford, what exactly does dist[v] represent, and what does the two-array (prev/curr) trick prevent?
